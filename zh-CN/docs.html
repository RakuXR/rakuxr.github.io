在现实世界中为你的 AI 加速。

通过六工具 MCP 接口面，从任意 AI 助手驱动 RakuAI 运行时。18ms 动到光延迟。亚毫米级空间精度。为眼镜而生，而非笔记本电脑。

为什么选 RakuAI？

5 项美国授权专利。优先权 2012 年 7 月。十年沉淀的知识产权，奠定了今天 AR 眼镜品类的基础。Claude、ChatGPT、Gemini 和 Copilot 都能驱动的运行时——多厂商设计。

为眼镜而生，而非笔记本电脑

18ms 动到光延迟。亚毫米级空间精度。在 Meta Quest 3、Xreal Air、Samsung Android XR 和 MiRZA AR2 上 OpenXR 一致。

世界模型编排

用于世界模型编排的六工具 MCP 接口面。每一帧、每个 NPC、每次交互都通过你选择的 AI 路由。Unity 的 MCP 面向设计期的场景操作——RakuAI 的 MCP 在运行期运作，逐帧进行。

工业级接口面

18 个原生 DLL。17 个路由模块。9,500+ 个 REST 端点。10,328 个绑定引擎函数。这些枯燥的数字，支撑起一个玩起来有趣的运行时。

快速上手

1

连接你的 AI 助手

将 RakuAI MCP 服务器接入 Claude Desktop（或任意支持 MCP 的客户端）。这六个工具会原生地出现在你的对话中。Claude Desktop 指南 →

2

驱动运行时

在任意环境中调用 get_metrics 或 get_scene_state 进行只读检视。在沙盒/开发环境中，可加载适配器、摄入帧、配置渲染目标并启动模拟。引擎始终是物理、碰撞和计分的权威。

关于六工具接口面的完整推理——为什么是这些工具、适配器接下来会解锁什么——都在博客里。阅读 MCP 构建日志 →

六工具 MCP 接口面

每个外部智能体都通过相同的六个工具访问引擎——load_world_model、ingest_frame、get_scene_state、set_render_target、start_simulation、get_metrics。标准契约，任意智能体。

受支持的 AI 客户端

接入 Claude Desktop → 获取自动配置脚本和完整的工具参考

Claude Desktop

原生支持 MCP。开发者门户中提供 PowerShell 自动配置脚本；适用于任意平台的手动 JSON 配置说明。

Cursor 与 IDE 客户端

任意支持 MCP 的编辑器或 IDE 都能通过相同的六工具接口面访问运行时。接入指南即将发布。

自定义智能体

在 MCP 边界上构建你自己的智能体。只读工具可在生产环境工作；变更类工具仅限沙盒/开发环境，默认拒绝。

自带 LLM

MCP 接口面与模型无关。ChatGPT 自定义 GPT、Gemini、Llama 和自研模型都讲同一套契约。

架构概览

引擎是确定性的权威。世界模型——视频预测器、神经辐射场、物理基础模型——是通过 MCP 适配器接入的贡献者。它们都不会凌驾于引擎对模拟中实际发生之事的权威之上。
