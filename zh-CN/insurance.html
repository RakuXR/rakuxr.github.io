面向房主、租户及其保险公司

一次手机扫描，一份可直接提交保险公司的物品清单。

RakuAI Capture 把一次 60–90 秒的房间手机扫描，转化为一份逐项物品清单 — 附带近似尺寸和预估重置价值 — 可直接交给你的保险公司。它为人们真正需要它的三个时刻而打造：提交理赔、规划搬家，以及精简物品。与 RakuAI 其余部分共用同一套 3D 高斯泼溅重建和 MCP 核心 — 无需新引擎。

本页面如实说明今日已上线的功能与路线图上的规划。我们记录一处空间并生成结构化清单；我们不是保险公司，也不设定保单条款或裁定理赔。

人们需要它的三个时刻。

我们刻意把房地产挂牌扫描让给了既有玩家（Matterport 等）。这是无人占据的面向消费者的切入点：一次自助扫描，在清单真正重要的时刻生成一份结构化物品清单。

理赔

遭受损失之后 — 火灾、盗窃、水浸、风暴 — 凭记忆重建你曾拥有的物品既痛苦又常引发争议。在投保或续保时做一次扫描，能在意外发生之前为你留下带时间戳的逐项记录。提交理赔从此变成"提交清单"，而不是"回忆你失去的一切"。

搬家

搬家公司、估值保障和运输保险都需要一份物品清单。每个房间扫描一次，即可生成带粗略尺寸的逐项明细 — 既便于搬家公司报价，也可记录上车前后的物品状况。

精简物品

清空一处住所 — 遗产处置、父母入住养老机构、孩子离家上大学 — 意味着要决定哪些留下、出售、捐赠或投保。一次扫描就能让全家共享一份逐项的房间物品视图。

一次扫描如何变成清单。

这正是贯穿产品其余部分的同一套 Raku Capture 流水线 — 扫描、重建、查询。保险只是对输出的一层封装，而非新引擎。

1

扫描房间

手持手机缓慢绕行一圈，60–90 秒，屏幕上有引导。无需专用硬件。

2

重建

帧 + 运动数据在云端重建为 3D 高斯泼溅 — 一个可探索的空间模型。

3

逐项清点

LLM 通过 MCP 接口遍历场景，列出可见物体，并为每件物品附上近似尺寸。

4

审核与导出

你编辑清单、确认价值，导出一份干净的清单交给你的保险公司。每个数值都标注为预估值。

输出长什么样。

一份示意性的单房间清单。真实输出由你的扫描生成，在提交给任何人之前都可完全编辑。

今天什么是真的，什么不是。

我们宁可把话说得保守些。以下是已上线与路线图之间的当前界线。

今日已上线

  • 基于手机的捕获与云端泼溅重建（即 Raku Capture 流程）。
  • 通过 MCP 接口的场景查询 — LLM 可遍历重建后的房间并描述其中的物品。
  • 多 LLM 支持：Claude、ChatGPT、Gemini 与 Copilot 都能驱动运行时。

推进中 / 路线图

  • 一份精致的单页可提交保险公司的导出（房间缩略图 + 逐项表格 + 合计 + 扫描元数据）。底层数据今日已存在；格式化文档正在构建中。
  • 针对特定保险公司的清单模板 — 由于不存在统一的行业标准格式，按单个试点处理。
  • 按扫描计费，以及面向消费者的变化追踪层级（每年重新扫描，查看差异）。

我们尚不具备的

  • SOC 2 认证。大型保险公司在采购前要求它；在这项就绪工作并行推进的同时，我们与 InsurTech 开展早期试点。
  • 任何已签署的保险公司合作。我们不声称获得任何保险公司的背书。
  • 评估鉴定权限。我们做记录和预估；我们不做估值或裁定。

面向保险公司与 InsurTech。

对保险公司而言，关键在于理赔费用与处理周期：在投保或续保时进行一次自助扫描，意味着日后更少的理赔员接触点。三种具体的合作方式。
