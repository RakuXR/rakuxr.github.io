工作原理

你的 AI 助手会说 MCP。RakuAI 就是这条连接另一端的运行时， 向眼镜、头显与浏览器驱动真实的空间输出。

17 个原生工具

完整的原生 MCP 接口：5 个只读 + 12 个变更。经 Schema 校验、审计日志记录、权限管控。加通用 api_call/api_search 透传至约 9,500 个 REST 端点。17 工具接口现已作为版本化公开合约（tools.v1.json，schema_version 1）固定，并配有防漂移测试，保证已发布接口与线上注册表保持一致。

5 分钟接入——只读模式

只要你的模型会说 MCP，它就会说 RakuAI。把现有 MCP 客户端指向运行时，即可在免费沙箱中获得完整的只读会话——5 个只读工具，无模型绑定，无引擎 SDK，无变更风险。

添加服务器 在你的 MCP 客户端配置中注册 raku-mcp（Claude Desktop 或任何支持 MCP 的助手）。
启动只读会话 只读模式是默认模式。5 个只读工具在所有模式下均可运行，包括生产环境。
读取实时场景状态 调用 get_scene_state 和 get_metrics，即可看到运行时响应。这立刻带来实际价值。
进阶至沙箱 12 个变更工具在沙箱/开发模式下解锁，可端到端驱动场景。

详细步骤：只读接入快速入门、LLM 集成指南、驱动场景实操演练。站内文档：Claude Desktop 指南。

审计与权限

什么能让安全负责人批准一个在空间中自主行动的 AI 代理？变更操作在设计上需要授权，每次调用都被记录，中继层强制执行分层限制。

默认拒绝 所有 12 个变更工具在沙箱/开发模式下默认拒绝；只有 5 个只读工具在所有模式下可用。生产环境中的变更操作需要经过明确、审计过的权限授予。
按会话导出审计日志 每次中继调用均被记录；账户所有者可导出其跨 /api/raku/* 接口的按会话审计记录，支持脱敏处理。
速率限制与配额 中继接口上提供免费与付费的速率限制和配额管控，确保用量有界，按套餐可追溯。

诚实说明：审计导出目前作用于中继范围与账户全局，尚不支持按连接器细分；配额执行在中继计数存储已配置的情况下生效。按连接器粒度和签名审计日志在路线图中。

对比 Unity MCP

Unity MCP（刚刚发布）：场景操作、资产摆放、 构建自动化——Claude Desktop 在设计时生成静态场景。

RakuAI MCP（自 2026 年 3 月起用于生产环境）：世界模型编排—— 任意 LLM 每一帧都将意图流入模拟步进。

Unity 是你的 AI 拿起的工具。RakuAI 则是你的 AI 栖身的身体。

为什么 MCP 原生很重要

  1. 无需自定义集成——只要你的模型会说 MCP，它就会说 RakuAI。大约五分钟内即可以只读模式接入。
  2. 从设计上即多厂商兼容——Claude、ChatGPT、Gemini、Copilot 都驱动同样的 17 个原生工具；兼容性在协议层保持一致。
  3. 版本化合约——17 工具接口已固定为 tools.v1.json，配有防漂移测试，你集成的接口不会悄悄改变。
  4. 可防御的权限体系——默认拒绝、审计日志、Schema 校验，以及供安全审查使用的按会话审计日志导出。
