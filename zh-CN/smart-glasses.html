一图看懂 MCP from glasses。

眼镜上的 LLM 通过 PWA 与 RakuAI 的 MCP 接口 对话 — 既可经由 stdio 桥接到配套设备，也可通过 HTTPS 连接到托管 MCP。运行时以空间原语作答：锚点、位姿、场景查询、泼溅句柄。

两端都是真实并已上线的。生产 MCP 服务器暴露 17 个原生工具（5 个只读 + 12 个改动）— 完整清单见 engine.html。其中约 12 个与眼镜相关，分为：6 个核心捕获／场景工具（在 MCP 页面有文档），以及 6 个 xr_ 适配器工具（会话、头部位姿、输入与锚点原语，用于眼镜驱动的流程）。托管 MCP 运行在美国东部的 Azure Container Apps 上。

我们不会承诺的：任何特定 OEM 会预装我们的 PWA、任何特定 LLM 厂商会默认通过我们路由，或我们无法控制的相机 API 会按我们期望的时间线出现。我们面向公开接口设计，把决定权留给平台。

权利要求覆盖了什么

  • 双模无线通信（低延迟 + 高带宽）
  • 智能眼镜 + 手机／算力主机架构
  • 从算力主机到眼镜的渲染卸载
  • 多人世界状态同步
  • 世界流送（场景增量传输，而非视频）

这些专利早于 Meta Ray-Bans（2021 年）、Xreal Air（2022 年）、Meta Ray-Ban Display（2024–25 年）以及 Android XR（2026 年）。对于希望在架构级权利要求上获得清晰前进路径的 OEM，我们在合作之外亦提供授权。
