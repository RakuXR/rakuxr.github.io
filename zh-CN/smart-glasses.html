三个阶段。一种授权模式。

市场进入按平台就绪度分阶段 — 而非按 RakuAI 的时间表。我们在你的设备已暴露的公开表面上与之相遇，待你准备好后再深化集成。

第 1 阶段 — 滩头阵地

Meta Ray-Ban Display PWA · 目标 2026 年第三季度

  • 入口：从眼镜的嵌入式浏览器安装的 PWA — 无应用商店门槛，无需在你的设备上安装 SDK
  • MCP 表面：17 个原生工具 + 6 个通用透传工具，对 Claude·ChatGPT·Gemini·Copilot 实时可用
  • 捕获：目前以配套手机作为相机；OEM 帧 API 落地后从眼镜直接捕获 (受硬件限制)
  • 架构：手机即算力（手机运行 AI/物理，眼镜负责显示 + 追踪）
  • 诚实标签：PWA 路径已达参考就绪；未与 Meta 达成联合营销或预装协议

第 2 阶段 — 平台

Android XR + 原生 SDK · 2027 年

  • 入口：同一 PWA 路径可推广至 Android XR 的标准化 Chrome 表面
  • 原生 SDK：raku-runtime 以带 OpenXR 兼容会话层的原生 DLL 形式 — 接入 OEM 的合成器、手部追踪管线或显示驱动
  • SDK 文档目标：合作伙伴级文档 2026 年第四季度；8–12 个开发周的集成路径
  • 合作伙伴：Samsung Galaxy XR + Android XR 被授权方；Xreal、Rokid、Vision Pro 浏览器同享分阶段方案
  • 诚实标签：Android XR 发布窗口取决于平台日程；无已签约 OEM

第 3 阶段 — 扩散

Xreal / Rokid / Galaxy XR / Vision Pro · 2027 年+

  • 通过浏览器路径支持长尾设备
  • 面向低规格眼镜的离线 / 间歇链路韧性
  • 可重复的自助资格 → PWA 试点 → SDK 管线
  • 为一家 Tier 3 战略合作伙伴打造联合营销发布应用
  • NVIDIA Inception 向 XR OEM 生态 BD 的友好引荐

NVIDIA Inception 网络：RakuAI 已加入 NVIDIA Inception（2026 年 5 月）。该网络提供向 XR 硬件生态的友好引荐 — 许多眼镜 OEM 及其芯片/光学供应商都是 Inception 成员或 NVIDIA 合作伙伴。

从眼镜到 MCP，一图说明。

眼镜上的 LLM 通过 PWA 与 RakuAI 的 MCP 表面对话 — 经由到配套设备的 stdio 桥，或经由 HTTPS 到托管 MCP。运行时以空间原语作答：锚点、姿态、场景查询、splat 句柄。

两端都是真实且已上线的。生产 MCP 服务器暴露 17 个原生工具（5 个只读 + 12 个变更）— 完整清单见 spatial-engine.html。其中与眼镜相关的约 12 个划分为：6 个核心捕获/场景工具（在 MCP 页面有文档），以及 6 个 xr_ 适配器工具（面向眼镜驱动流程的会话、头部姿态、输入和锚点原语）。托管 MCP 运行在 East US 的 Azure Container Apps 上。
