面向经纪人、经纪机构与房源团队

把任意房源变成可漫游的 3D 空间。

用手机扫描一处房产 — 无需 300 美元的专业设备，也无需特殊相机 — RakuAI 即可将其重建为可漫游的 3D 高斯泼溅。随后买家便能做到视频漫游做不到的事：向房源提问 — "厨房中岛有多宽？""主卧朝哪个方向？" — 由你或你的门户已在使用的任意 LLM 通过我们的 MCP 接口作答。

这是 RakuAI 的旗舰垂直领域，而我们正以开放的方式构建它。下方页面会如实区分今日已上线的功能与路线图上的规划。我们尚未产生收入，正在寻找抢先体验的经纪人和试点经纪机构 — 而非宣称已有成品。

从手机走查到一处买家可以对话的房源。

驱动平台其余部分的同一套 Raku Capture 流水线 — 一步一步来。使用箭头、圆点或键盘切换。

第 1 步

用手机扫描

拿着早已放在口袋里的手机走查房产即可。无需三脚架，无需 300 美元的捕获设备，无需特殊相机，也无需培训日。

第 2 步

重建为 3D 泼溅

画面帧与运动数据在云端被重建为 3D 高斯泼溅 — 一个平滑、趋近照片级真实的空间模型，而非拼接的全景图。

第 3 步

买家在浏览器中漫游

分享一个链接即可。买家在任意设备上按自己的节奏穿行于这个空间 — 无需头显，无需安装应用，也无需预约看房。

第 4 步

向房源提任何问题

通过我们的 MCP 接口，LLM 基于真实扫描作答 — 尺寸、朝向、某个房间里有什么。多厂商兼容：Claude、ChatGPT、Gemini 或 Copilot 都能驱动它。

它如何运作。

从你手中的手机到一处可分享、可提问的房源，只需三步。这正是贯穿 RakuAI 的同一套 Raku Capture 核心 — 房地产只是对输出的一层封装，而非新引擎。

1

扫描

在屏幕引导下手持手机走查住宅。每个空间仅需几分钟，无需任何你尚未拥有的设备。

2

重建并发布

RakuAI 将扫描重建为可漫游的 3D 高斯泼溅，并给你一个分享链接，可直接放入你的房源、邮件或门户。

3

漫游并提问

买家在浏览器中穿行空间，并用自然语言向它提问 — 由任意 LLM 通过 MCP 接口、基于扫描内容作答。

这与 Matterport 漫游有何不同。

3D 房源漫游并不新鲜。新鲜的是：用一部手机就能做到、让空间能够回答问题，而且永不把你锁定在单一厂商上。

典型 3D 漫游（如 Matterport） 面向房地产的 RakuAI
捕获硬件 通常需要专用相机或约 300 美元以上的设备，或付费的扫描技师上门 你早已拥有的一部手机 — 无需特殊相机
重建方式 全景图 / 网格漫游 3D 高斯泼溅 — 导航更平滑、更趋近照片级真实
向空间提问 原生不支持 — 它是一个浏览器，而非问答接口 支持 — 可通过 MCP 由 LLM 查询，基于扫描内容作答
由哪个 AI 作答 不适用，或单一厂商 多厂商 — Claude、ChatGPT、Gemini 或 Copilot，由你选择
厂商锁定 绑定在单一平台与浏览器上 中立、基于标准的 MCP 接口；自带你自己的模型

该对比反映的是公开可见的典型工作流程，而非基准测试；其他产品的能力和价格会发生变化。我们指出我们正在构建的差异点，并在下方如实说明哪些尚未完成。

经纪人与经纪机构为何看重它。

卖点不是"更花哨的漫游"。而是更少的无效看房、能预先筛选买家的房源，以及低到可以用在每一处房源上的捕获成本 — 而不仅仅是豪宅。

每一处房源，而非只有豪宅

当捕获变成一次手机走查、而非付费的扫描技师上门时，沉浸式漫游就不再是只为高端住宅保留的预算项目。其边际成本趋近于零，因此可以用在你的全部房源上。

更少的低意向看房

能够漫游空间并预先得到基本问题答案的买家，往往会自我筛选。你真正安排的线下看房，对象都是已经在虚拟中走查过的人。

一处凌晨两点也能作答的房源

"车库能停下两辆车吗？""卧室朝哪个方向？"房源会基于扫描内容自行回答这些问题，无论买家何时在浏览 — 而不只是在工作时间。

你的 AI，你的门户

由于它是一个中立的 MCP 接口，经纪机构或门户可以把房源问答接入它早已在付费使用的任意 LLM，并以自有品牌呈现 — 无需被迫切换厂商。

今天什么是真的，什么不是。

我们宁可把话说得保守些。RakuAI 尚未产生收入，这个垂直领域还很早期。以下是已上线与路线图之间的诚实界线。

今日已上线

  • 基于手机的捕获与云端 3D 高斯泼溅重建（即 Raku Capture 流程）。
  • 通过 MCP 接口的场景查询 — LLM 可遍历重建后的空间并回答关于它的问题。
  • 多 LLM 支持：Claude、ChatGPT、Gemini 与 Copilot 都能驱动运行时。

推进中 / 路线图

  • 一个精致的、专为房地产打造的房源浏览器，以及为买家、经纪人和门户调优的一键分享链接。
  • 房源级问答模板（尺寸、朝向、固定装置、"包含哪些"）以及面向门户和类 MLS 站点的可嵌入小组件。
  • 经纪人/经纪机构账户、按房源组织管理，以及关于买家提问内容的分析。

我们尚不具备的

  • 付费的房地产客户。我们尚未产生收入，正在寻找抢先体验的经纪人和试点经纪机构 — 我们不声称拥有并不存在的签约客户。
  • 一个今天就能开启的 MLS/门户集成；目前的集成是按试点逐个推进的工作。
  • 测量保证。LLM 从扫描中报告的尺寸是为方便买家而提供的预估值，而非测绘或评估鉴定。
