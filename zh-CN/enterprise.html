企业空间 AI

一次性空间捕获。任何 LLM 都能调用的 MCP 运行时 — 17 个原生工具、约 147 个透传工具、约 9,500 个 API 端点。为企业纵向整合而打造：保险（理赔与物品清单）、零售与设施（空间规划、变更检测）以及机器人（持久空间记忆）— 并诚实标注哪些已正式发布（GA），哪些在路线图中。

企业纵向整合

三个垂直领域，一套空间捕获 + MCP 基底。保险与机器人是已获资金支持的活跃切入点。零售与设施以需求驱动方式服务 — 当设计合作伙伴确定后即升级为获资金支持的切入点。

三者均运行在同一 Raku Capture 流水线 + MCP 工具接口上： api.rakuai.com.

保险

活跃切入点

待办任务： 理赔员或房主用手机扫描房间。AI 助手提取物品清单、物品价值与损坏证据 — 无需现场技术人员。

  • ✓基于手机的splat捕获（GA）
  • ✓面向理赔 AI 的 MCP 查询接口（GA）
  • ✓物品清单模板
  • ◦重大损失 / 结构检查 — 路线图

部署拓扑

边缘设备将数据流式传输至 RakuAI 运行时；你的控制平面管理权限、审计与集群策略 — 全域默认拒绝。

大规模部署空间 AI

滑动浏览企业部署的样貌 — 在单一运行时上的扩展性、安全与治理。

部署

边缘 + 运行时设备将数据流式传输至 RakuAI 运行时；控制平面与数据路径保持分离。
双设备手机作算力、眼镜作显示 — 原生设计，而非外挂拼接。
OpenXR基于标准的渲染，实现广泛的头显兼容性。

扩展

全集群策略一套策略适用于全域，而非逐台设备配置。
18ms 目标在所有受支持硬件上维持动到光延迟目标。
热感知动态质量回退使设备保持在其工作范围内。

安全与治理

默认拒绝除非策略明确允许，否则任何操作都不会运行。
RBAC + SSO/SCIM基于角色的访问控制，集成企业身份系统。
不可变审计记录每一项决策，用于合规与事件审查。

  • 私有 / 品牌 MCP 端点 — 第 2 阶段
  • 数据驻留选项 — 第 3 阶段
  • 自定义品牌 / OEM 套件 — 第 2 阶段
  • 合同 SLA — 第 2 阶段

第 1 阶段 = 正在构建。第 2–3 阶段 = 已获资金的路线图，尚未 GA。暂无 SOC 2 报告；就绪评估进行中。

诚实现状： 捕获 + 查询已投入生产。SSO/SCIM、默认拒绝 RBAC、审计日志、自定义 MCP 命名空间、私有/品牌端点、数据驻留及 OEM 打包均为尚未 GA 的路线图项目。今天要求 SOC 2 的采购方将得到诚实的“就绪评估进行中”答复。没有虚构客户，没有尚无值班支持的 SLA。当你联系我们时，我们会准确告知有哪些功能可用。
