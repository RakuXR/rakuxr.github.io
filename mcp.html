How it works

Your AI assistant speaks MCP. RakuAI is the runtime on the other end of that connection, driving real spatial output to glasses, headsets, and the browser.

The Six Tools

The full MCP surface. Schema-validated, audit-logged, permission-gated.

Tool What it does Permission
load_world_model(adapter_name, config) Register a world model adapter Sandbox/dev only
ingest_frame(adapter_name, frame_data, frame_index, timestamp) Push a frame into scene graph Sandbox/dev only
get_scene_state(include_physics, include_transforms) Read-only snapshot All modes
set_render_target(target_type, config) Configure output (WebGL/VR/native/offscreen) Sandbox/dev only
start_simulation(tick_rate, max_duration, realtime) Begin simulation loop Sandbox/dev only
get_metrics() Performance telemetry All modes

vs Unity MCP

Unity MCP (just released): scene manipulation, asset placement, build automation — Claude Desktop generates static scenes at design time.

RakuAI MCP (production since March 2026): world-model orchestration — any LLM streams intent into the simulation step every frame.

Unity is the tool your AI picks up. RakuAI is the body your AI inhabits.
