샘플 MCP 통합

현실 세계에서 AI를 강화하세요.

6개 도구 MCP 표면 위에서 동작하는 RakuAI 런타임을 위한 스타터 프로젝트. AI 어시스턴트를 연결한 다음, 실제 엔진을 상대로 반복하세요. 설계부터 멀티 벤더 — Claude, ChatGPT, Gemini, Copilot.

통합 패턴

아래 각 패턴은 외부 에이전트가 MCP를 통해 런타임을 구동하는 한 가지 방법을 보여줍니다 — 동일한 6개 도구(load_world_model, ingest_frame, get_scene_state, set_render_target, start_simulation, get_metrics)가 읽기 전용 검사부터 완전한 시뮬레이션 제어까지 모든 것을 다룹니다. 읽기 전용 패턴은 프로덕션에서 동작하며, 변경 패턴은 샌드박스/개발 환경 전용으로 기본 거부됩니다.

읽기 전용 검사

Claude Desktop, Cursor, 커스텀 GPT, 사내 에이전트 등 모든 클라이언트에서 get_metrics와 get_scene_state를 사용하세요. 프로덕션을 포함한 모든 환경에서 안전합니다.

프로덕션 안전

어댑터 로딩

load_world_model로 월드 모델 백엔드(비디오 예측기, 뉴럴 래디언스 필드, 물리 파운데이션 모델, 콘텐츠 팩)를 등록하세요. 어댑터는 표준 계약을 따르고, 엔진은 권한을 유지합니다.

샌드박스 / 개발

프레임 수집

ingest_frame으로 생성형 모델 프레임을 씬 그래프에 푸시하세요. 비주얼은 어댑터에서 나오고, 게임플레이(물리, 충돌, 점수)는 엔진에 남습니다. 양쪽은 MCP 경계를 통해서만 서로를 인식하면 됩니다.

샌드박스 / 개발

렌더 타깃 구성

set_render_target은 월드가 렌더링되는 위치를 구성합니다: WebGL, VR 헤드셋, 네이티브 윈도우, 오프스크린. 같은 씬, 같은 권한, 다른 표면.

샌드박스 / 개발

시뮬레이션 제어

start_simulation은 구성 가능한 틱 레이트로 시뮬레이션 루프를 시작합니다. 설계상 프로덕션에서는 거부됩니다 — 외부 에이전트는 요청할 수는 있지만 명령할 수는 없습니다. 샌드박스와 개발 환경에서는 이를 허용합니다.

샌드박스 / 개발

커스텀 에이전트 구성

당신의 에이전트에서 필요한 대로 6개 도구를 조합하세요. MCP 서버는 호출자별, 도구별로 기본 거부 권한을 적용하고 전체 감사 로그를 기록합니다. 경계를 관통하지 말고, 경계 위에서 구축하세요.

프로덕션 안전 (읽기 전용)

지원되는 AI 클라이언트

MCP를 지원하는 모든 클라이언트가 런타임에 접근할 수 있습니다. 현재: Claude Desktop(연결 가이드 제공 중). 진행 중: Cursor, 에이전트 모드의 GitHub Copilot, MCP HTTP 전송을 통한 커스텀 GPT. MCP 표면은 설계상 모델 비종속적입니다 — 모든 프레임, 모든 NPC, 모든 상호작용이 당신이 선택한 AI를 통해 라우팅됩니다.
