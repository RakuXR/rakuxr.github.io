로그인 EN

당신의 AI에게 현실 세계의 몸을 부여하세요.

RakuAI는 여러분의 모델이 공간 월드를 구축하거나, 시뮬레이션하거나, 상호작용해야 할 때 머무는 런타임입니다. MCP로 연결하세요. 커스텀 통합 불필요. 설계부터 멀티 벤더 — 여러분의 모델은 통합 깊이가 아니라 품질로 경쟁합니다.

MCP로 연결하기 → 아키텍처 읽기 →

LLM 개발사에게 MCP 네이티브가 중요한 이유

1. 모델이 그냥 동작합니다.

여러분의 어시스턴트가 MCP를 말한다면, RakuAI도 말합니다. 도구 6개. 모든 모델에 동일한 계약. 커스텀 통합 불필요.

2. 설계부터 멀티 벤더.

Claude, ChatGPT, Gemini, Copilot — 모두 같은 방식으로 RakuAI를 구동합니다. 여러분의 모델은 의도의 품질로 경쟁합니다.

3. 프로덕션 기본 거부.

감사 로그 기록. 스키마 검증. 읽기 전용 도구는 프로덕션에서 동작하고, 변경 도구는 샌드박스/개발 환경에서 동작합니다.

아키텍처

가능해지는 것들

  • 말하기 전에 관찰하는 NPC — 여러분의 모델이 매 프레임 씬 상태를 읽습니다
  • 실제 GPS 좌표를 중심으로 재구성되는 경험
  • 시뮬레이션 스텝 위의 월드 모델 — 여러분의 모델은 콘텐츠 공급자가 아니라 두뇌입니다
  • 크로스 플랫폼 전달 — 스마트 글래스, 헤드셋, 웹, 모바일

vs Unity

Unity는 막 Native MCP를 출시했습니다. 설계 시점 씬 조작에는 좋습니다. Claude Desktop이 씬을 만들고 Unity가 렌더링합니다. RakuAI의 MCP는 시뮬레이션 시점에 동작합니다. 여러분의 모델은 씬 파일을 생성하는 것이 아니라 — 프레임 단위로 월드를 구동합니다. 다른 카테고리이며, 둘 다 유효합니다.

어댑터 계약

6개 핵심 도구를 넘어, 파트너는 어댑터를 제공합니다:

  • VeoEnvironmentAdapter — Google의 Veo
  • RunwayEnvironmentAdapter — Runway
  • YourModelAdapter — 여러분의 모델

어댑터 계약은 공개되어 있습니다. 프레임 + 씬 델타를 생성하는 어떤 모델이든 어댑터가 될 수 있습니다.

멀티 벤더 증거

저희 개발 루프는 4개 AI 벤더를 병렬로 사용합니다:

  • Claude Code가 런타임의 대부분을 작성합니다
  • Gemini가 PR을 리뷰합니다
  • ChatGPT가 아키텍처를 함께 고민합니다
  • GitHub Copilot이 나머지가 놓친 것을 잡아냅니다

저희는 단일 모델에 충성하지 않습니다. 저희 런타임도 마찬가지입니다.
