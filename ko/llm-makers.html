로그인 EN

당신의 AI에게 현실 세계의 몸을 부여하세요.

RakuAI는 여러분의 모델이 MCP를 통해 머무는 추론 전용 공간 런타임입니다 — 씬 그래프, 물리, 시뮬레이션 스텝, 렌더 타깃. 하나의 JSON-Schema 계약. 모든 벤더. 학습도, 가중치도, 통합 비용도 없습니다. MCP로 연결하면 여러분의 모델은 통합 깊이가 아니라 의도의 품질로 경쟁합니다.

LLM 개발사에게 MCP 네이티브가 중요한 이유

1. 한 번 연결하면, 모든 것을 구동합니다.

여러분의 어시스턴트가 MCP를 말한다면, RakuAI도 말합니다. 버전 관리되고 안정적인 하나의 JSON-Schema 계약: 네이티브 도구 17개(읽기 5개 + 변경 12개)에 더해 api.rakuai.com에서 제공되는 ~9,500개 중 ~147개의 패스스루 엔드포인트. 커스텀 통합도, 벤더별 포크도 없습니다.

2. 설계부터 멀티 벤더.

계약은 Claude, ChatGPT, Gemini, Copilot에 대해 동일합니다 — 구조적으로 동일합니다. 독점 없음. 벤더별 종단 간 적합성 테스트가 진행 중이며, 어떤 벤더에 대해서도 검증된 동등성을 주장하기 전에 결과를 공개하겠습니다.

3. 추론 전용. 학습하지 않습니다.

RakuAI는 여러분의 모델이 추론 시점에 호출하는 공간 런타임입니다. 우리는 어떤 모델 가중치도 보유하지 않고, 여러분의 데이터로 학습 작업을 실행하지 않으며, 어떤 것도 파인튜닝하지 않습니다. 여러분의 데이터는 여러분의 것입니다. 이것은 확고한 원칙입니다.

4. 프로덕션에서 기본 거부.

읽기 도구(get_scene_state, list_objects, get_metrics, get_world_model, get_render_target)는 프로덕션에서 동작합니다. 변경 도구(set_render_target, start_simulation 외 10개)는 기본적으로 샌드박스화되고 감사 로그가 기록됩니다.

1단계 — 커넥터 + 첫 연구소 확보

우리는 연구소 파트너십 전략의 1단계에 있습니다. 목표는 한 연구소의 도구 디렉터리에 일류 MCP 커넥터로 등재되고, 재현 가능한 종단 간 데모를 함께 실행하는 것입니다. 오늘 서명된 연구소 파트너는 없습니다 — 그것은 현황이 아니라 목표입니다.

MCP로 연결

MCP를 말하는 어떤 어시스턴트도 오늘 api.rakuai.com에 연결할 수 있습니다. 17개 도구 계약은 안정적이고 버전 관리됩니다. 여러분의 커넥터 디렉터리에 등재하면 우리가 유지보수합니다.

레퍼런스 데모 실행

Raku Capture로 실제 공간을 스캔한 다음, MCP를 통해 여러분의 모델로 씬을 질의하고 조작하세요. 데모는 재현 가능하며 양쪽 모두 독점 도구 없이 실행됩니다.

통합 공동 마케팅

공동 블로그 게시물, 공유 커넥터 등재, 그리고 사이트 내 "LLM 통합 가이드". 우리가 배포 작업을 하고, 여러분은 가리킬 수 있는 공간 역량 스토리를 얻습니다.

Anthropic 우선, 그 다음 개방

Anthropic Claude Startups 멤버십(2026년 5월 승인)이 가장 따뜻한 첫 경로입니다. NVIDIA Inception(2026년 5월 승인)은 신뢰성과 컴퓨팅 관계를 더합니다. 우리는 OpenAI, Google, Microsoft 생태계와 적극적으로 대화를 열고 있습니다.

아키텍처

가능해지는 것들

  • 말하기 전에 관찰하는 NPC — 여러분의 모델이 매 프레임 씬 상태를 읽습니다 (현재 가능)
  • 시뮬레이션 스텝 위의 월드 모델 — 여러분의 모델은 콘텐츠 공급자가 아니라 두뇌입니다 (현재 가능)
  • 크로스 플랫폼 전달 — 스마트 글래스, 헤드셋, 웹, 모바일 (현재 가능)
  • 실제 GPS 좌표를 중심으로 재구성되는 경험 (로드맵)

vs Unity

Unity는 막 Native MCP를 출시했습니다. 설계 시점 씬 조작에는 좋습니다. Claude Desktop이 씬을 만들고 Unity가 렌더링합니다. RakuAI의 MCP는 시뮬레이션 시점에 동작합니다. 여러분의 모델은 씬 파일을 생성하는 것이 아니라 — 프레임 단위로 월드를 구동합니다. 다른 카테고리이며, 둘 다 유효합니다.

어댑터 계약 (로드맵)

계획된 확장 표면: 파트너는 모델별 출력을 RakuAI 씬 계약으로 변환하는 어댑터를 제공합니다. 어댑터 인터페이스는 현재 설계 중이며, 아직 라이브 어댑터는 없습니다.

  • VeoEnvironmentAdapter — 계획: Google의 Veo (로드맵)
  • RunwayEnvironmentAdapter — 계획: Runway (로드맵)
  • YourModelAdapter — 여러분의 모델, 계약이 출시되면

어댑터 계약 설계가 진행 중입니다. 프레임 + 씬 델타를 생성하는 어떤 모델이든 어댑터가 될 수 있습니다. 구현을 요청하기 전에 사양을 공개하겠습니다.

실제 멀티 벤더

우리의 개발 루프는 매일 — 네 개의 AI 벤더를 병렬로 사용합니다:

  • Claude Code가 우리 런타임의 대부분을 작성합니다
  • Gemini가 PR을 리뷰합니다
  • ChatGPT가 아키텍처를 함께 점검합니다
  • GitHub Copilot이 나머지가 놓친 것을 잡아냅니다

우리는 어떤 단일 모델에도 충성하지 않습니다. 우리 런타임도 마찬가지입니다.

솔직한 현황: MCP 계약은 모든 벤더에 대해 구조적으로 동일합니다. 벤더별 종단 간 적합성 테스트 — 월드 로드, 상태 읽기, 조작, 왕복 검증 — 는 아직 완료되지 않은 2단계 산출물입니다. 결과를 공개하겠습니다. "멀티 벤더"를 오늘 검증된 동등성의 주장으로 받아들이지 마세요.

함께하기

우리는 첫 연구소 커넥터 파트너를 적극적으로 찾고 있습니다.

Anthropic OpenAI Google Meta xAI Mistral Cohere Stability AI

이들은 우리가 목표로 하는 연구소이며 — 기존에 서명한 파트너가 아닙니다. 독점 없음: 멀티 벤더 계약이 곧 제품입니다. 여러분이 파운데이션 모델이나 어시스턴트 제품을 구축하고 있고 — 공간 런타임 팀을 꾸리지 않고 — MCP를 통해 진짜로 측정 가능한 씬을 모델에 부여하고 싶다면, 이야기를 나누고 싶습니다.

구체적으로: 우리는 여러분의 커넥터를 등재하고, 여러분의 플랫폼용 LLM 통합 가이드를 제공하며, 재현 가능한 레퍼런스 데모를 함께 실행하겠습니다. MCP 연결은 오후 한나절이면 됩니다. 공간 스토리는 첫날부터 여러분의 것입니다.
