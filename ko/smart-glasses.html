스마트 글래스 OEM 기술 리드를 위해

스마트 글래스 OEM이 직접 구축하지 않고 라이선스하는 공간 런타임.

RakuAI는 OEM이 직접 구축하지 않고 라이선스하는 LLM 벤더 중립 공간 런타임입니다. 1단계 진입점: Meta Ray-Ban Display PWA — 글래스의 브라우저에서 설치 가능한 웹 앱, Claude·ChatGPT·Gemini·Copilot이 구동할 수 있는 MCP 표면, 그리고 헤드 착용형 디스플레이에 맞춘 발열/지연 예산. 2단계 로드맵: Android XR + 네이티브 SDK(2026년 3분기–2027년). 우리는 여러분의 하드웨어와 경쟁하지 않습니다 — 그 위의 AI 경험을 더 빨리 출시하도록 돕습니다.

2026–2027년에 글래스를 출시하는 Meta, Samsung, Google, Xreal, Rokid 등의 엔지니어링 및 파트너십 리드를 위해 작성되었습니다. 무엇이 출시되었고, 무엇이 레퍼런스 전용이며, 무엇이 하드웨어에 의존하는지 솔직하게 밝힙니다. 서명된 OEM 파트너는 없습니다.

시장 진입은 RakuAI의 일정이 아니라 — 플랫폼 준비도에 따라 단계화됩니다. 우리는 여러분의 기기가 이미 노출하는 공개 표면에서 만나고, 준비가 되면 통합을 심화합니다.

1단계 — 진입점

Meta Ray-Ban Display PWA · 목표 2026년 3분기

  • 진입: 글래스의 임베디드 브라우저에서 설치 가능한 PWA — 앱 스토어 게이팅 없음, 기기에 SDK 설치 불필요
  • MCP 표면: 네이티브 도구 17개 + 범용 패스스루 도구 6개, Claude·ChatGPT·Gemini·Copilot에 대해 라이브
  • 캡처: 현재는 컴패니언 폰을 카메라로 사용; OEM의 프레임 API가 적용되면 글래스에서 직접 캡처 (하드웨어 의존)
  • 아키텍처: 폰을 컴퓨트로(폰이 AI/물리 실행, 글래스는 디스플레이 + 트래킹)
  • 솔직한 라벨: PWA 경로는 레퍼런스 준비 완료; Meta와의 공동 마케팅이나 사전 설치는 합의되지 않음

2단계 — 플랫폼

Android XR + 네이티브 SDK · 2027년

  • 진입: 동일한 PWA 경로가 Android XR의 표준화된 Chrome 표면 전반에 일반화됨
  • 네이티브 SDK: OpenXR 호환 세션 계층을 갖춘 네이티브 DLL로서의 raku-runtime — OEM의 컴포지터, 핸드 트래킹 파이프라인, 디스플레이 드라이버에 연결
  • SDK 문서 목표: 파트너급 문서 2026년 4분기; 8–12 개발 주(週) 통합 경로
  • 파트너: Samsung Galaxy XR + Android XR 라이선시; Xreal, Rokid, Vision Pro 브라우저도 동일한 단계별 제안
  • 솔직한 라벨: Android XR 출시 구간은 플랫폼 일정에 따라 달라짐; 서명된 OEM 없음

3단계 — 팬아웃

Xreal / Rokid / Galaxy XR / Vision Pro · 2027년+

  • 브라우저 경로를 통한 롱테일 기기 지원
  • 저사양 글래스를 위한 오프라인 / 간헐적 링크 복원력
  • 반복 가능한 셀프 자격 → PWA 파일럿 → SDK 파이프라인
  • Tier 3 전략 파트너 한 곳을 위한 공동 마케팅 출시 앱
  • XR OEM 생태계 BD로의 NVIDIA Inception 우호적 소개

NVIDIA Inception 네트워크: RakuAI는 NVIDIA Inception에 합류했습니다(2026년 5월). 그 네트워크는 XR 하드웨어 생태계로의 우호적 소개를 제공합니다 — 많은 글래스 OEM과 그들의 실리콘/광학 공급사가 Inception 멤버이거나 NVIDIA 파트너입니다.

OEM 통합, 단계별로.

첫 대화부터 공동 출시 경험까지의 경로를 스와이프해 보세요. SDK 락인 없음 — 우리는 여러분의 기기가 이미 노출하는 공개 웹 표면에서 만납니다.

1

PWA로 연결

RakuAI는 여러분 기기의 웹 표면에서 PWA로 설치됩니다 — 앱 스토어 게이팅 없음. 글래스가 rakuai.com/capture-app/를 로드하면 런타임이 라이브됩니다.

  • 설치 가능한 웹 앱 진입, 오늘
  • WebXR / WebGPU 클라이언트 경로
  • 시작에 네이티브 셸 불필요
2

MCP를 통해 어떤 LLM으로든 구동

Claude, ChatGPT, Gemini, Copilot이 17개 네이티브 MCP 도구를 통해 런타임을 구동합니다 — 컴패니언 기기로의 stdio 브리지 또는 호스팅 MCP로의 HTTPS 경유.

  • 17개 네이티브 MCP 도구 (읽기 전용 5개 + 변경 12개)
  • 글래스 흐름을 위한 6개 xr_ 어댑터 도구
  • 벤더 중립 — 착용자가 LLM을 선택
3

착용자의 공간 캡처

프레임 + IMU가 클라우드 재구성으로 흐르고 수 분 내에 탐색 가능한 가우시안 스플랫으로 돌아옵니다 — 현재는 컴패니언 폰에서, 카메라 API가 적용되면 글래스에서 직접 (하드웨어 의존: OEM의 프레임 소스 API 필요).

  • GLOMAP + Brush 재구성, .spz 출력
  • 컴패니언 폰 캡처 현재 출시
  • 2026년 3분기 경로의 글래스 직접 캡처
4

발열 한계에 맞추기

폰을 컴퓨트로 사용하면 글래스가 시원하게 유지됩니다: 폰이 물리와 AI를 실행하고, 글래스는 렌더링과 트래킹을 하며, 링크는 영상이 아니라 씬 델타를 전달합니다. 세 가지 발열 모드가 기기에 적응합니다.

  • 18 ms 모션-투-포톤 목표
  • ~20 Hz 씬 델타 동기화, 400–1400 B
  • 퍼포먼스 / 밸런스드 / 저전력 모드
5

NDA 하에 공동 출시

상호 NDA로 xr_ 어댑터 도구, 글래스 캡처 프리뷰, WebGPU 스플랫 파이프라인에 접근 — 여러분 기기를 위한 공동 마케팅 출시 경험으로의 경로 포함.

  • NDA 하의 OEM SDK 프리뷰
  • 아키텍처 수준 특허 라이선싱 제공 가능
  • 몇 주가 아니라 며칠 안에 상호 NDA

글래스에서의 MCP, 다이어그램 한 장으로.

글래스의 LLM은 PWA를 통해 RakuAI의 MCP 표면과 대화합니다 — 컴패니언 기기로의 stdio 브리지 또는 호스팅 MCP로의 HTTPS 경유. 런타임은 공간 프리미티브로 응답합니다: 앵커, 포즈, 씬 질의, 스플랫 핸들.

양쪽 모두 실제이며 출시되어 있습니다. 프로덕션 MCP 서버는 네이티브 도구 17개(읽기 전용 5개 + 변경 12개)를 노출합니다 — 전체 목록은 spatial-engine.html 참조. 그중 글래스 관련 ~12개는 다음과 같이 나뉩니다: 6개 핵심 캡처/씬 도구(MCP 페이지에 문서화됨), 그리고 6개 xr_ 어댑터 도구(글래스 구동 흐름을 위한 세션, 헤드 포즈, 입력, 앵커 프리미티브). 호스팅 MCP는 East US의 Azure Container Apps에서 실행됩니다.

로드맵, 솔직하게.

세 열, 세 타임라인. 우리는 출시할 수 없는 것을 약속하지 않으며, 우리가 제어하지 않는 플랫폼 동작에 의존하는 것을 표시합니다.
