AI 어시스턴트에서 RakuAI를 구동하세요.

17개 네이티브 MCP 도구(읽기 전용 5개 + 변형 12개)와 universal api_call/api_search 패스스루로 ~9,500개 REST 엔드포인트에 접근. 2026년 3월부터 프로덕션 배포. 기본 거부, 전체 감사 로그.

빌드 로그 읽기 → GitHub에서 보기 →

작동 방식

여러분의 AI 어시스턴트는 MCP를 말합니다. RakuAI는 그 연결의 반대편에 있는 런타임으로, 안경, 헤드셋, 브라우저로 진짜 공간 출력을 구동합니다.

17개 네이티브 도구

완전한 네이티브 MCP 표면: 읽기 전용 5개 + 변형 12개. 스키마 검증, 감사 로그 기록, 권한 게이트 적용. 추가로 universal api_call/api_search 패스스루로 ~9,500개 REST 엔드포인트 접근 가능.

도구 기능 권한
load_world_model(adapter_name, config) 월드 모델 어댑터 등록 샌드박스/개발 전용
ingest_frame(adapter_name, frame_data, frame_index, timestamp) 씬 그래프에 프레임 푸시 샌드박스/개발 전용
get_scene_state(include_physics, include_transforms) 읽기 전용 스냅샷 모든 모드
set_render_target(target_type, config) 출력 구성 (WebGL/VR/네이티브/오프스크린) 샌드박스/개발 전용
start_simulation(tick_rate, max_duration, realtime) 시뮬레이션 루프 시작 샌드박스/개발 전용
get_metrics() 성능 텔레메트리 모든 모드

5분 만에 연결 — 읽기 전용

모델이 MCP를 지원하면 RakuAI와 통신할 수 있습니다. 기존 MCP 클라이언트를 런타임에 연결하고 무료 샌드박스에서 읽기 전용 세션(읽기 전용 도구 5개)을 바로 시작하세요. 모델 종속 없음, 엔진 SDK 없음, 변형 위험 없음.

서버 추가 MCP 클라이언트 설정(Claude Desktop 또는 MCP 지원 어시스턴트)에 raku-mcp를 등록합니다.
읽기 전용 세션 시작 읽기 전용 모드가 기본값입니다. 읽기 전용 도구 5개는 프로덕션 포함 모든 모드에서 실행됩니다.
라이브 씬 상태 읽기 get_scene_state와 get_metrics를 호출해 런타임 응답을 확인하세요. 바로 실용적인 가치를 제공합니다.
샌드박스로 업그레이드 변형 도구 12개는 샌드박스/개발 모드에서 잠금 해제되어 씬을 엔드투엔드로 구동할 수 있습니다.

단계별 가이드: 읽기 전용 연결 퀵스타트, LLM 통합 가이드, 씬 구동 워크스루. 사이트 내: Claude Desktop 가이드.

멀티 벤더 동등성

호출자가 Claude, ChatGPT, Gemini, Copilot 중 어느 것이든 컨트랙트는 동일합니다 — 이것이 핵심입니다. 각 벤더의 17개 네이티브 도구 및 패스스루 지원 현황을 추적하는 동등성 매트릭스를 공개합니다. 동등성은 프로토콜 수준에서 유지되며, 클라이언트별 스키마 처리는 공개 아티팩트로 투명하게 추적됩니다.

벤더 네이티브 도구 17개 패스스루 상태
Claude프로토콜 수준프로토콜 수준공개됨
ChatGPT프로토콜 수준프로토콜 수준공개됨
Gemini프로토콜 수준프로토콜 수준공개됨
Copilot프로토콜 수준프로토콜 수준공개됨

솔직한 참고사항: 매트릭스는 공개됨 상태이며 독립적으로 검증된 것은 아닙니다. 공개 기록상 전체 17개 도구 + 패스스루 적합성 점검을 통과한 벤더는 아직 없습니다 — 클라이언트별 엔드투엔드 검증은 매트릭스 자체에서 추적되는 오픈 Phase 1 작업입니다.

감사 및 권한 정책

보안 담당자가 공간에서 작동하는 자율 에이전트를 승인할 수 있게 하는 것은 무엇일까요? 변형은 설계상 게이팅되고, 모든 호출이 기록되며, 릴레이는 단계별 제한을 강제합니다.

기본 거부 변형 도구 12개는 기본적으로 샌드박스/개발 전용입니다. 읽기 전용 도구 5개만 모든 모드에서 실행됩니다. 프로덕션 변형에는 명시적이고 감사된 권한 부여가 필요합니다.
세션별 감사 로그 내보내기 모든 릴레이 호출이 기록됩니다. 소유 계정은 /api/raku/* 서피스 전반에 걸친 세션별 감사 추적을 내보낼 수 있으며 리덕션 안전합니다.
속도 제한 & 할당량 릴레이 서피스에서 무료/유료 릴레이 속도 제한 및 할당량 배관이 작동하여, 사용량이 티어별로 제한되고 책임추적 가능합니다.

솔직한 참고사항: 감사 내보내기는 현재 릴레이 범위 및 계정 전체이며, 아직 커넥터별 세분화는 지원되지 않습니다. 할당량 적용은 릴레이의 카운터 저장소가 구성된 경우에 활성화됩니다. 커넥터별 세분화 및 서명된 감사 로그는 로드맵입니다.

vs Unity MCP

Unity MCP (막 출시됨): 씬 조작, 에셋 배치, 빌드 자동화 — Claude Desktop이 설계 시점에 정적 씬을 생성합니다.

RakuAI MCP (2026년 3월부터 프로덕션): 월드 모델 오케스트레이션 — 어떤 LLM이든 매 프레임 시뮬레이션 스텝으로 의도를 스트리밍합니다.

Unity는 여러분의 AI가 집어 드는 도구입니다. RakuAI는 여러분의 AI가 머무는 몸입니다.

MCP 네이티브가 중요한 이유

  1. 커스텀 통합 불필요 — 여러분의 모델이 MCP를 말한다면, RakuAI도 말합니다.
  2. 설계부터 멀티 벤더 — Claude, ChatGPT, Gemini, Copilot 모두 동일한 17개 네이티브 도구를 구동합니다.
  3. 프로덕션 준비 완료 — 기본 거부, 감사 로그 기록, 스키마 검증.

현재 출시된 기능 vs 로드맵

출시된 것과 출시되지 않은 것을 명확히 표시합니다 — 정직한 레이블링은 기능이지, 각주가 아닙니다.

  • 커넥터별 감사 세분화 및 서명된 감사 로그

  • 인프로덕트 권한 단계 UI; 범위가 지정된 권한 프로필
  • 멀티 리전 릴레이, SSO/SCIM, 지연시간 SLO
