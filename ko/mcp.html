AI 어시스턴트에서 RakuAI를 구동하세요.

MCP 도구 6개. 2026년 3월부터 프로덕션 배포. 기본 거부 권한. 전체 감사 로그. 여러분의 AI가 공간 월드를 구축하거나, 조회하거나, 시뮬레이션해야 할 때 대화하는 런타임입니다.

빌드 로그 읽기 → GitHub에서 보기 →

작동 방식

여러분의 AI 어시스턴트는 MCP를 말합니다. RakuAI는 그 연결의 반대편에 있는 런타임으로, 안경, 헤드셋, 브라우저로 진짜 공간 출력을 구동합니다.

6가지 도구

전체 MCP 표면. 스키마 검증, 감사 로그 기록, 권한 게이트 적용.

도구 기능 권한
load_world_model(adapter_name, config) 월드 모델 어댑터 등록 샌드박스/개발 전용
ingest_frame(adapter_name, frame_data, frame_index, timestamp) 씬 그래프에 프레임 푸시 샌드박스/개발 전용
get_scene_state(include_physics, include_transforms) 읽기 전용 스냅샷 모든 모드
set_render_target(target_type, config) 출력 구성 (WebGL/VR/네이티브/오프스크린) 샌드박스/개발 전용
start_simulation(tick_rate, max_duration, realtime) 시뮬레이션 루프 시작 샌드박스/개발 전용
get_metrics() 성능 텔레메트리 모든 모드

vs Unity MCP

Unity MCP (막 출시됨): 씬 조작, 에셋 배치, 빌드 자동화 — Claude Desktop이 설계 시점에 정적 씬을 생성합니다.

RakuAI MCP (2026년 3월부터 프로덕션): 월드 모델 오케스트레이션 — 어떤 LLM이든 매 프레임 시뮬레이션 스텝으로 의도를 스트리밍합니다.

Unity는 여러분의 AI가 집어 드는 도구입니다. RakuAI는 여러분의 AI가 머무는 몸입니다.

MCP 네이티브가 중요한 이유

  1. 커스텀 통합 불필요 — 여러분의 모델이 MCP를 말한다면, RakuAI도 말합니다.
  2. 설계부터 멀티 벤더 — Claude, ChatGPT, Gemini, Copilot 모두 동일한 6개 도구를 구동합니다.
  3. 프로덕션 준비 완료 — 기본 거부, 감사 로그 기록, 스키마 검증.
