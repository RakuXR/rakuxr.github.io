The MCP tool surface

RakuAI ships a Model Context Protocol server (raku-mcp) that any MCP-compatible LLM client can connect to - Claude Desktop, Cursor, custom agents. There are 17 native tools in the production server (Wave 7Z, May 2026), plus six universal passthrough tools.

Scene state 2 tools

  • get_scene_stateSnapshot the current scene graph.
  • capture_sceneCapture a render-time observation (color, depth, segmentation).

Scene mutation 4 tools

  • add_scene_nodeInsert an entity into the scene graph.
  • update_scene_nodeChange properties on an existing node.
  • set_node_transformPosition / rotation / scale.
  • remove_scene_nodeDelete a node.

World-model adapters 3 tools

  • load_world_modelBind an adapter (Runway, Veo, Marble, Genie, content-pack).
  • ingest_frameFeed a world-model output frame into the runtime.
  • set_render_targetChoose where rendered frames go.

XR session 6 tools

  • xr_get_sessionInspect headset / glasses session state.
  • xr_set_head_poseDrive head pose (sim, recording, or remote).
  • xr_simulate_inputInject controller / hand-tracking events.
  • xr_list_anchorsList spatial anchors in the session.
  • xr_create_anchorAnchor a node to world coordinates.
  • xr_remove_anchorRelease an anchor.

Telemetry & sim 2 tools

  • get_metricsFrame time, draw calls, memory, tool-call latency.
  • start_simulationStep physics + world model under sandbox mode.

Universal passthrough 6 tools

  • api_search / api_describe / api_callDiscover and invoke any of ~9,500 raku-api REST endpoints.
  • engine_search / engine_describe / engine_callDiscover and invoke any callable engine function (~10,328).

Source: raku-runtime/src/mcp/raku_mcp_server.py (READ_ONLY_TOOLS + MUTATION_TOOLS = 17 unique) and raku-runtime/src/mcp/tools/_catalog.py.
