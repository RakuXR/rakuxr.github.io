Govern an autonomous agent

Before a security owner lets an AI agent drive a spatial runtime, they need three guarantees: the agent can do nothing by default, a permission is a declarative rule a human can read and approve, and every permitted action leaves a tamper-evident record. This demo runs the same agent action through all three states against the live RakuAI API.

1 · Denied

The action is evaluated against the deny-by-default policy. It matches no allow rule, so it is refused. Nothing runs.

—

2 · Permitted

A security owner adds a declarative allow rule (policy-as-code, not a code change). The same action now matches and is permitted.

—

3 · Attested

The permitted decision is wrapped in an HMAC-signed, hash-chained record, then re-verified. Mutate one field and the signature rejects it.

—


  

    The govern engine lives in raku-api at
    services/govern/; this page calls
    GET /api/v1/govern/demo. The signing key never leaves the server — the
    record carries only the non-secret key id and the HMAC. When no signing key is
    configured the attestation is an honest signed=false envelope (the hash
    chain still proves integrity and ordering), never a fabricated signature.
