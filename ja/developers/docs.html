API ドキュメント

RakuAI API の完全リファレンスです。すべてのエンドポイントは /api/v1/ 配下にバージョン管理されています。

クイックスタート

初めてのゲームを 3 ステップで生成：

1. API キーを取得

開発者 NDA に同意し、ダッシュボード から API キーを作成してください。

2. 最初の API コール

cURL curl -X POST https://api.rakuai.com/api/v1/generate \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d '{"prompt": "A platformer where you jump between clouds"}'
Python import httpx

resp = httpx.post(
  "https://api.rakuai.com/api/v1/generate",
  headers={"Authorization": "Bearer YOUR_API_KEY"},
  json={"prompt": "A platformer where you jump between clouds"},
)
game = resp.json()
print(game["raku_config"]["game"]["title"])
JavaScript const resp = await fetch("https://api.rakuai.com/api/v1/generate", {
  method: "POST",
  headers: {
    "Content-Type": "application/json",
    "Authorization": "Bearer YOUR_API_KEY",
  },
  body: JSON.stringify({ prompt: "A platformer where you jump between clouds" }),
});
const game = await resp.json();
console.log(game.raku_config.game.title);

3. .raku 設定を使う

レスポンスには raku_config オブジェクトが含まれます — これは Raku ランタイムが読み込んでプレイできる、完全なゲーム設定です。

認証

すべての API リクエストには、Authorization ヘッダーに Bearer トークンが必要です。ログイン・登録から得られる JWT アクセストークン、または開発者ダッシュボードから取得した API キーのいずれかを使用できます。

Authorization: Bearer <your_token_or_api_key>

POST /api/v1/generate

POST /api/v1/generate

自然言語プロンプトからゲーム設定を生成します。

パラメータ型必須説明
promptstring必須ゲームの説明（1〜500 文字）

レスポンス: game_id、raku_config、genre、生成メタデータを返します。

POST /api/v1/iterate

POST /api/v1/iterate

追加の指示で既存のゲーム設定を改良します。

パラメータ型必須説明
game_idstring必須改良対象のゲーム ID
messagestring必須改良指示（1〜1000 文字）
current_configobject必須変更する現在の .raku 設定

Games

GET /api/v1/games

保存済みゲームをページネーション付きで一覧表示します。

GET /api/v1/games/{id}

特定ゲームの完全な設定を取得します。

DELETE /api/v1/games/{id}

保存済みゲームを完全に削除します。

GET /api/v1/usage

GET /api/v1/usage

生成・改良回数、ティア情報、残りクォータを取得します。

API キー管理

POST /api/v1/developer/keys

指定したスコープで新しい API キーを作成します。

GET /api/v1/developer/keys

すべての API キー（有効・失効済み）を一覧表示します。

DELETE /api/v1/developer/keys/{key_id}

API キーを失効させます。取り消せません。

レート制限

API リクエストはユーザーのティアごとにレート制限されます：

ティア生成 / 日改良 / 日同時実行
匿名301
Free10301
Pro1005003
Enterprise無制限無制限10

制限を超えると 429 Too Many Requests と Retry-After ヘッダーが返ります。
