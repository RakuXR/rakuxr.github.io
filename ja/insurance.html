持ち家・賃貸の方とその保険会社向け

スマホスキャン 1 回で、保険会社に提出できる家財目録を。

RakuAI Capture は、部屋を 60〜90 秒スマホでスキャンするだけで、その家財の明細リスト — おおよその寸法と推定再調達価格付き — に変換します。保険会社にそのまま渡せるリストです。人が実際にそれを必要とする 3 つの場面のために構築しました。保険金請求、引っ越しの計画、そしてダウンサイジングです。RakuAI のほかの製品と同じ 3D ガウシアンスプラット再構成と MCP コアを使用 — 新しいエンジンは不要です。

このページは、今日出荷できるものとロードマップ上にあるものについて正直に伝えます。私たちは空間を記録し、構造化された目録を作成します。私たちは保険会社ではなく、保険契約の条件を定めたり保険金請求を解決したりすることはありません。

メンバー： 詳しく見る ↓
キャプチャフローを試す → 保険会社 & InsurTech パートナーシップ →

これが必要になる 3 つの場面。

私たちは不動産物件リスティング向けのキャプチャを、あえて既存事業者（Matterport などの企業）に譲りました。これは、誰も押さえていないコンシューマー向けのくさび（差別化点）です。目録が実際に重要となる場面のために、構造化された家財目録を生成する、セルフサービスのスキャンです。

保険金請求

損害が発生したあと — 火災、盗難、水濡れ、嵐 — 自分が何を所有していたかを記憶から再構成するのは苦痛で、争いも頻繁に起きます。加入時や更新時にスキャンしておけば、何かが起きる前に、タイムスタンプ付きの明細記録が手に入ります。請求は「失ったものをすべて思い出す」のではなく「目録を提出する」ことになります。

引っ越し

引っ越し業者、評価額補償、運送保険のいずれも、家財リストを求めます。部屋ごとに 1 回スキャンすれば、おおよその寸法付きの明細マニフェストが生成されます — 引っ越し業者の見積もりや、トラック輸送の前後で状態を記録するのに役立ちます。

ダウンサイジング

家を片づけること — 遺品整理、親の介護施設への入居、子どもの大学進学による独立 — は、何を残し、売り、寄付し、保険をかけるかを決めることを意味します。スキャンすれば、部屋に何があるかについて、家族全員が共有できる明細ビューが 1 つ手に入ります。

1 回のスキャンが、どのように目録になるのか。

これは、製品全体で使われているのと同じ Raku Capture パイプラインです — スキャン、再構成、クエリ。保険は出力をラップしたものであり、新しいエンジンではありません。

1

部屋をスキャン

スマホを持ってゆっくり 1 周します。60〜90 秒、画面でガイドされます。特別なハードウェアは不要です。

2

再構成

フレーム + モーションがクラウドで 3D ガウシアンスプラットに再構成されます — 探索可能な空間モデルです。

3

明細化

LLM が MCP サーフェス経由でシーンを巡回し、見えているオブジェクトを、品目ごとのおおよその寸法とともにリスト化します。

4

確認 & エクスポート

リストを編集し、価格を確認して、保険会社に渡せるきれいな目録をエクスポートします。すべての価格には「見積もり」と明記されます。

出力はこんな見た目です。

部屋 1 室分の目録の一例です。実際の出力はあなたのスキャンから生成され、誰かに提出する前に自由に編集できます。

品目 おおよそのサイズ 推定再調達価格
3 人掛けソファ210 × 90 cm$1,200
55″ テレビ123 × 71 cm$650

本棚（本入り）80 × 200 cm$380

上記の数値はすべて見積もりであり、鑑定評価ではありません。物体認識や価格推定は完璧ではありません — どこかに提出する前に、ご自身でリストを確認・修正してください。RakuAI はあなたの補償内容を定めたり、保険金の支払額を決めたり、再調達価格を保証したりすることはありません。それはあなたの保険会社が行います。

これに、なぜ RakuAI なのか。

キャプチャ市場は混み合っていますが、コンシューマー向けのセルフサービススキャン → 構造化された目録というパイプラインは、大きく空いています。

手法 強み 家財目録としての弱点
スマホ写真のリスト（DIY / メール） 無料で、誰でもできる 寸法なし、構造なし、紛失しやすく、損害後の再構成が苦痛
損害査定員の訪問 権威がある 総コストは 1 回の訪問あたり約 200〜500 ドル。保険金請求が提出されたあとにしか行われない
Matterport 型の 3D ウォークスルー 高精細なキャプチャ 不動産物件リスティング向けに構築 — 家財の明細出力も、保険会社向けのエクスポートもない
RakuAI Capture セルフサービスのスマホスキャン、スプラットの精細さ、MCP 経由の LLM 駆動による明細化 正直な但し書き：価格は見積もりであり、正式な保険会社向けエクスポート形式はまだ構築中

今日できること、まだできないこと。

私たちは控えめに約束したいと考えています。これが、出荷済みとロードマップを分ける現在の線引きです。

今日出荷済み

  • スマホによるキャプチャとクラウドでのスプラット再構成（Raku Capture フロー）。
  • MCP サーフェス経由のシーンクエリ — LLM が再構成された部屋を巡回し、何があるかを説明できます。
  • マルチ LLM 対応：Claude、ChatGPT、Gemini、Copilot のいずれもランタイムを駆動できます。

進行中 / ロードマップ

  • 洗練された 1 ページの保険会社提出用エクスポート（部屋のサムネイル + 明細表 + 合計 + スキャンメタデータ）。元になるデータは今日存在し、整形された文書を構築中です。
  • 保険会社ごとの目録テンプレート — 業界標準の単一フォーマットが存在しないため、パイロットごとの作業として対応します。
  • スキャン単位の従量課金と、コンシューマー向けの変更追跡ティア（毎年再スキャンして差分を確認）。

まだ持っていないもの

  • SOC 2 認証。大手保険会社は調達前にこれを要求します。私たちは、その準備作業を並行して進めながら、InsurTech との初期パイロットを実施しています。
  • 締結済みの保険会社パートナーシップ。いかなる保険会社からの推奨も主張しません。
  • 鑑定評価の権限。私たちは記録し見積もるだけで、評価額を定めたり示談したりはしません。

保険会社・InsurTech の皆様へ。

保険会社にとっての本質は、損害調査費用とサイクルタイムです。加入時や更新時のセルフサービススキャンは、その後の査定員との接点を減らします。具体的な 3 つの関わり方をご紹介します。

1. 機能に関するお問い合わせ

家財キャプチャレイヤーを評価しており、RakuAI が御社の製品、希望する目録フォーマット、オンボーディングフローにどう対応するかについて、書面での回答をご希望の場合。御社の制約をお知らせください。今日実現できることと、パイロットに向けて当社が構築するものをお返しします。

partnerships@rakuai.com →

2. 90 日間の無料パイロット

四半期につき 1 社のパートナー。御社のオンボーディングにスキャンフローを統合し（ホワイトラベルまたはサイドカー型ウェブアプリ）、約 100 件の顧客スキャンを収集し、ケーススタディを共同で公開します。パイロットに費用は請求しません。UX チームへのアクセスと公開の許可をお願いします。

パイロットについて相談する →

SOC 2 が取得されるまで、初期パイロットは保険会社向けレビューポータルを備えた RakuAI クラウド上で実施されます。

3. InsurTech 統合

御社はすでに保険金請求のオーケストレーション、空撮・外観データ、自動車損害査定などを手がけているものの、屋内の家財に関する仕組みがない場合。私たちは、御社が自社ブランドで API 経由で再販できる、キャプチャ・明細化レイヤーになれます。
