AI アシスタントから RakuAI を駆動する。

17 のネイティブ MCP ツール（読み取り専用 5 + ミューテーション 12）と universal api_call/api_search パススルーで約 9,500 の REST エンドポイントにアクセス可能。2026 年 3 月から本番稼働中——バージョン管理されたパブリックツールコントラクト、5 分で接続できる読み取り専用パス、デフォルト拒否・監査ログ済みの権限付き。

ビルドログを読む → GitHub で見る →

仕組み

あなたの AI アシスタントは MCP を話します。RakuAI はその接続の反対側にあるランタイムで、 グラス・ヘッドセット・ブラウザへ本物の空間出力を駆動します。

17 のネイティブツール

完全なネイティブ MCP インターフェース：読み取り専用 5 + ミューテーション 12。スキーマ検証済み、監査ログ記録、権限ゲート付き。さらに universal api_call/api_search パススルーで約 9,500 の raku-api REST エンドポイントにアクセス可能。17 ツールのサーフェスはバージョン管理されたパブリックコントラクト（tools.v1.json、schema_version 1）としてピン留めされており、公開済みサーフェスがライブレジストリから乖離した場合に CI を失敗させるアンチドリフトテスト付き。

5 分で接続 — 読み取り専用

あなたのモデルが MCP を話せるなら、RakuAI も話せます。既存の MCP クライアントをランタイムに向けるだけで、無料サンドボックス上で読み取り専用セッション——5 つの読み取り専用ツール——が動作します。モデルロックインなし、エンジン SDK なし、ミューテーションリスクなし。

サーバーを追加 MCP クライアントの設定（Claude Desktop または MCP 対応のアシスタント）に raku-mcp を登録します。
読み取り専用セッションを開始 読み取り専用モードがデフォルトです。5 つの読み取り専用ツールは、本番を含む全モードで動作します。
ライブシーンの状態を読み取る get_scene_state と get_metrics を呼び出して、ランタイムの応答を確認します。これだけで即座に価値が得られます。
サンドボックスへ昇格 12 のミューテーションツールはサンドボックス／開発でアンロックされ、シーンをエンドツーエンドで操作できます。

ステップバイステップ: 読み取り専用クイックスタート、LLM 統合ガイド、シーン操作ウォークスルー。サイト内: Claude Desktop ガイド。

マルチベンダーパリティ

呼び出し元が Claude、ChatGPT、Gemini、Copilot のいずれであっても、コントラクトは同一です——これが差別化ポイントです。各ベンダーを 17 のネイティブツール + パススルー全体に対して追跡するパリティマトリックスを公開しています。パリティはプロトコルレベルで成立し、クライアント固有のスキーマ処理はオープンなリビングドキュメントとして追跡されています。

監査と権限の仕組み

空間内で動作する自律エージェントをセキュリティ責任者が承認できるのはなぜか？ミューテーションは設計上ゲートされ、すべての呼び出しはログ記録され、リレーが段階的な制限を強制します。

デフォルト拒否 12 のミューテーションツールはすべてデフォルトでサンドボックス／開発のみ。5 つの読み取り専用ツールだけが全モードで動作します。本番でのミューテーションには明示的な監査済み権限付与が必要です。
セッションごとの監査ログエクスポート すべてのリレー呼び出しが記録され、オーナーアカウントは /api/raku/* サーフェス全体にわたるセッションごとの監査証跡をエクスポートできます（リダクション対応）。
レート制限とクォータ 無料・有料プランのリレーレート制限とクォータ管理がリレーサーフェスで機能し、ティアごとに使用量を制限・追跡します。

正直な注記：監査エクスポートは現在リレースコープでアカウント全体適用であり、コネクタごとの粒度には対応していません。クォータ強制はリレーのカウンターストアが設定されている場合に有効です。コネクタごとの粒度と署名付き監査ログはロードマップです。

Unity MCP との比較

Unity MCP（リリースされたばかり）：シーン操作、アセット配置、 ビルド自動化 — Claude Desktop が設計段階で静的シーンを生成します。

RakuAI MCP（2026 年 3 月から本番稼働）：ワールドモデルのオーケストレーション — 任意の LLM が毎フレーム、シミュレーションステップにインテントをストリームします。

Unity はあなたの AI が手に取るツールです。RakuAI はあなたの AI が宿る身体です。

契約は 17 のネイティブツールです。アダプターはパートナーが接続するための手段です。

アダプターコントラクトは公開されています。現在の実績あるアダプターはファーストパーティ／リファレンス実装です。幅広いサードパーティアダプターエコシステムはロードマップの目標であり、現状ではありません。

MCP ネイティブが重要な理由

  1. カスタム統合不要 — あなたのモデルが MCP を話せば、RakuAI も話せます。読み取り専用モードには約 5 分で接続できます。
  2. 設計段階からマルチベンダー — Claude、ChatGPT、Gemini、Copilot がすべて同じ 17 のネイティブツールを対象にしており、パリティはプロトコルレベルで成立しています。
  3. バージョン管理されたコントラクト — 17 ツールのサーフェスは tools.v1.json としてピン留めされており、アンチドリフトテストにより、統合対象が黙って変更されることはありません。
  4. 説明責任のある権限 — デフォルト拒否、監査ログ記録、スキーマ検証済み、セキュリティレビュー用のセッションごとの監査ログエクスポート付き。
