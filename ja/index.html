なぜ Unity / Unreal ではなく RakuAI なのか

ゲームエンジンは、デザイナーが手に取る「ツール」として作られました。RakuAI は、AI が宿る「身体」として作られています。

Unity との違い

Unity MCP（リリースされたばかり）：デザイン時のシーン操作。RakuAI MCP（2026年3月から本番稼働）：実行時の、毎フレームのワールドモデルオーケストレーション。Unity はあなたの AI が手に取るツール。RakuAI はあなたの AI が宿る身体です。

Unreal との違い

スマートグラスの熱・電力エンベロープに合わせて設計。デスクトップエンジンから後付けで移植したものではありません。Unreal には RTX 4090 が必要です。RakuAI に必要なのは Wi-Fi 7 とスマートフォンだけです。

ワールドモデルのためのユニバーサルランタイム

AI ワールドモデルは、リアルタイムでコヒーレントな 3D 環境を生成できるようになりました — しかし、ランタイムは提供されません。
RakuAI はそれを提供します。 ワールドモデルの出力から、プレイ可能で埋め込み可能な体験までのギャップを埋めます。

📥

Ingest（取り込み）

あらゆるワールドモデルの出力 — Genie 3、Runway GWM-1、Veo、Marble、または独自モデル — を、単一のアダプターインターフェースで取り込み。SceneGraph、DepthMap、VideoFrame、Gaussian Splat、すべて対応。

⚙️

Simulate（シミュレート）

任意のワールドモデル出力の上に、物理、ゲームロジック、NPC AI、空間音響、マルチプレイヤーを追加。Bullet3 物理。XAudio2 空間化。オンデバイス LLM 推論。すべて同じパイプライン内で。

🖥️

Render（レンダリング）

Win32 ネイティブ、WebGL/WebGPU、OpenXR ヘッドセット、または任意のカスタムサーフェスへ出力。同じシーングラフがどこでもレンダリング。エンジン乗り換え不要、移植不要、再ビルド不要。

ファーストクラスの OpenXR サポート。ワールドモデルの出力を Quest 3、Varjo、HoloLens、その他任意の OpenXR ヘッドセットへ直接レンダリング。追加のミドルウェアなしで空間インタラクションが可能。

本番環境対応

徹底的にテスト済み。コンパクトなフットプリント。何千もの本番用 API 関数。スタンドアロンネイティブバイナリとして出荷 — VM なし、マネージドランタイムのオーバーヘッドなし。

REST API

全ランタイムサブシステムをカバーする 17 のルーターモジュール、9,500 以上のエンドポイントを備えた完全な REST API。ワールド作成、物理シミュレーション、AI エージェント制御、状態観測のすべてを認証付き HTTP 呼び出しで実行 — DLL リンク不要。

他と違う理由

AI ネイティブアーキテクチャ

動的難易度調整、プレイヤープロファイリング、適応音響、NPC パーソナリティがエンジンコアに組み込まれています。プラグインではありません。常時オン。

即時実行

.raku ファイルは構造化されたゲーム定義です。エンジンがリアルタイムに解釈します。ファイルを編集し、ゲームを更新。コンパイル不要。

オンデバイス AI

AI はあなたのデバイス上でローカルに実行されます。クラウド呼び出しなし。プライバシーファースト。オフライン動作。パーソナライゼーションが即座に行われます。

XR 対応

自然な入力と空間インタラクションを備えた完全な XR サポート。すべてのゲームはデフォルトで XR 対応。フラグ 1 つで切り替え可能。

空間音響

真の 3D 位置オーディオ。適応音楽はゲームプレイの強度に応じて反応。エンジンに組み込まれた没入型サウンドデザイン。

開発者にやさしい

任意の AI アシスタントでゲームを作成。無料で使用可能。ライセンス料なし。ロイヤリティなし。
