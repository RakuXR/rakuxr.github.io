エンタープライズ向け空間 AI

ワンパスの空間キャプチャ。あらゆる LLM が呼び出せる MCP ランタイム — 17 個のネイティブツール、約 147 個のパススルーツール、約 9,500 個の API エンドポイント。エンタープライズの業種ロールアップ向けに構築：保険（請求 & 家財目録）、リテール & ファシリティ（スペース計画、変化検出）、ロボティクス（永続的な空間メモリ）— 何が GA で何がロードマップ上かを正直に表記。

メンバー： 詳細 ↓

エンタープライズ業種ロールアップ

3 つの業種、ひとつの空間キャプチャ + MCP 基盤。保険とロボティクスはアクティブな資金提供済みウェッジです。リテール & ファシリティはリアクティブに対応 — デザインパートナーがコミットすると資金提供済みウェッジに昇格します。

3 つすべてが api.rakuai.com 上の同じ Raku Capture パイプライン + MCP ツールサーフェスで動作します。

保険

アクティブウェッジ

解決すべき仕事： 鑑定人または住宅所有者がスマートフォンで部屋をスキャンします。AI アシスタントが家財目録、品目の価値、損害の証拠を抽出 — 現場技術者は不要です。

  • ✓スマホベースの Splat キャプチャ（GA）
  • ✓請求 AI 向け MCP 照会サーフェス（GA）
  • ✓家財目録テンプレート
  • ◦大規模損害 / 構造検査 — ロードマップ
保険プラットフォームの詳細 →

リテール & ファシリティ

リアクティブ / インバウンド

解決すべき仕事： 店舗スタッフや施設管理者が部屋や通路をスキャンします。AI アシスタントがフィット確認、配置、変化検出のクエリに回答 —「このソファは窓際に収まる？」や「先月から何が変わった？」— 3D パイプラインチームなしで。

  • ✓スマホベースの Splat キャプチャ（GA）
  • ✓MCP 照会サーフェス（GA）
  • ◦カタログ統合 & 配置ツール — ロードマップ
  • ◦マルチサイト変化検出 — ロードマップ
リテール / ファシリティのパイロットを依頼 →

ロボティクス

アクティブウェッジ

解決すべき仕事： 自律システムが環境の空間メモリをキャプチャし、継続的に更新します。AI アシスタント（またはロボットのプランナー）がリアルタイムで照会 — 手作りの地図も CAD も不要です。

  • ✓永続的な空間メモリ基盤（GA）
  • ✓ロボットプランナー向け MCP 照会サーフェス（GA）
  • ◦リアルタイム増分更新 — ロードマップ
  • ◦マルチロボットフリート連携 — ロードマップ

デプロイメントトポロジー

エッジデバイスが RakuAI ランタイムへストリーミング。あなたのコントロールプレーンが権限、監査、フリートポリシーを統制 — 全資産にわたって deny-by-default。

デプロイメント

エッジ + ランタイムデバイスが RakuAI ランタイムへストリーミング。コントロールプレーンはデータパスから分離されたまま。
デュアルデバイス電話 = コンピュート、グラス = ディスプレイ — 後付けではなくネイティブ。
OpenXR標準ベースのレンダリングで幅広いヘッドセット互換性。

スケール

フリート全体のポリシーひとつのポリシーがデバイスごとではなく全資産に適用されます。
18 ms 目標対応ハードウェア全体でモーション・トゥ・フォトン目標を維持。
サーマル対応動的な品質フォールバックでデバイスを動作範囲内に保ちます。

セキュリティ & ガバナンス

Deny-by-defaultポリシーが明示的に許可しない限り、何も実行されません。
RBAC + SSO/SCIMエンタープライズ ID 統合によるロールベースアクセス。
改ざん不能な監査すべての判断をコンプライアンスとインシデント検証のために記録。

エンタープライズプリミティブ

あらゆる空間キャプチャ業種が必要とする共有プラットフォーム — データレジデンシー、SSO、プライベート / ブランド MCP エンドポイント、OEM は制限付きロードマップ項目であり、GA ではありません。Capture + Query は本日利用可能です。Governance & Identity、Custom MCP / OEM、データレジデンシーはフェーズごとに明確に表記されています。

業種とユースケースをお知らせください — パイロット提案をカスタマイズします。パイロットを依頼 ↓

Capture + Query  GA

ワンパスの電話キャプチャから照会可能な空間モデルへ。あらゆる LLM が結果に対して MCP ツールサーフェスを呼び出せます。

  • スマホベースの Splat キャプチャ
  • MCP ツールサーフェス（照会 / 実行）
  • 業種テンプレート：保険、リテール、ロボティクス
  • プロジェクトごとの空間スコープ
  • API キー / トークン認証（GA）

Governance & Identity  ロードマップ

エンタープライズ調達に必要な認証、監査、フリートポリシーの層。フェーズ 1 で構築中 — まだ本番環境にはありません。

  • SSO（OIDC / SAML）— フェーズ 1
  • SCIM プロビジョニング — フェーズ 1
  • Deny-by-default RBAC — フェーズ 1
  • AI アクションの監査ログ — フェーズ 1
  • SOC 2 準備状況評価 — 進行中

Custom MCP & OEM  ロードマップ

テナントごとのカスタム MCP 名前空間（あなたのシステムを当社のものと並べて呼び出し可能）とプライベート / ホワイトラベルエンドポイント。フェーズ 2 の目標。

  • テナントごとの MCP ツール名前空間 — フェーズ 2
  • プライベート / ブランド MCP エンドポイント — フェーズ 2
  • データレジデンシーオプション — フェーズ 3
  • カスタムブランディング / OEM バンドル — フェーズ 2
  • 契約上の SLA — フェーズ 2

フェーズ 1 = アクティブな構築。フェーズ 2–3 = 資金提供済みロードマップ、まだ GA ではありません。SOC 2 レポートはまだなし。準備状況評価は進行中。

正直なステータス： Capture + Query は本番環境にあります。SSO/SCIM、deny-by-default RBAC、監査ログ、カスタム MCP 名前空間、プライベート / ブランドエンドポイント、データレジデンシー、OEM バンドルはすべてまだ GA ではないロードマップ項目です。本日 SOC 2 を求める調達には「準備状況評価は進行中」という正直な回答をお返しします。捏造した顧客なし、オンコールでまだ裏付けのない SLA なし。ご連絡いただければ、何が利用可能かを正確にお伝えします。
