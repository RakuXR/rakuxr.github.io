ドキュメント

6 つのツールから成る MCP サーフェスを通じて、任意の AI アシスタントから RakuAI ランタイムを駆動。

はじめに

1

AI アシスタントを接続

RakuAI MCP サーバーを Claude Desktop（または任意の MCP 対応クライアント）に組み込みます。6 つのツールが会話の中でネイティブに表示されます。Claude Desktop ガイド →

2

ランタイムを駆動

任意の環境で読み取り専用の検査を行うには get_metrics や get_scene_state を呼び出します。サンドボックス／開発環境ではアダプターのロード、フレームの取り込み、レンダーターゲットの設定、シミュレーションの開始ができます。物理・衝突・スコアリングの権威はエンジンが保持します。

3

ビルドログを読む

6 つのツールの設計理由 — なぜこのツール、アダプターが次に何を解き放つのか — はブログで解説しています。MCP ビルドログを読む →

6 つのツールから成る MCP サーフェス

外部エージェントはすべて同じ 6 つのツール — load_world_model、ingest_frame、get_scene_state、set_render_target、start_simulation、get_metrics — を通じてエンジンに到達します。標準コントラクト、任意のエージェント。

対応 AI クライアント

Claude Desktop セットアップガイド →

Claude Desktop

RakuAI MCP サーバーを設定ファイルに追加すれば、6 つのツールが Claude の会話内でネイティブに表示されます。

Cursor

Cursor の MCP 統合経由で、IDE 内から直接ランタイムを駆動。コードを書きながらシーン状態を検査できます。

ChatGPT (OpenAI)

カスタム GPT または Plugin として MCP サーバーを接続。同じ 6 ツールでランタイムを駆動可能。

GitHub Copilot

Copilot Chat の MCP 拡張により、IDE 内から RakuAI ランタイムを呼び出せます。

ランタイムコントラクト

ランタイムは決定論的な権威として動作します。エージェントはツールを呼び出してワールドモデルをロードし、フレームを供給し、状態を読み取ります。物理・衝突・スコアリング・マルチプレイヤー状態はエンジンが管轄します。

完全な MCP コントラクトとスキーマ仕様は 「6 つの MCP ツール」解説を参照してください。

API リファレンス

レンダリング、物理、オーディオ、AI、XR、ネットワーキング、アニメーションなどをカバーする包括的な REST API。 MCP サーフェスはこの API レイヤーの上に構築されています。
