ドキュメント

17 ツールのネイティブ MCP サーフェスを通じて、任意の AI アシスタントから RakuAI ランタイムを駆動。

RakuAI を選ぶ理由

米国特許 5 件取得済み。優先日 2012 年 7 月。今日の AR グラスカテゴリーを支える 10 年来の知的財産。Claude、ChatGPT、Gemini、Copilot のいずれもが駆動するランタイム — 設計段階からマルチベンダー対応。

ノートPCではなくグラスのために設計

モーション・トゥ・フォトン遅延 18ms。サブミリメートルの空間精度。Meta Quest 3、Xreal Air、Samsung Android XR、MiRZA AR2 で OpenXR 準拠。

ワールドモデルのオーケストレーション

ワールドモデルのオーケストレーションのための 17 ツールのネイティブ MCP サーフェス。すべてのフレーム、すべての NPC、すべてのインタラクションを、お好みの AI 経由でルーティング。Unity の MCP は設計時のシーン操作を対象とします — RakuAI の MCP は実行時に、毎フレーム動作します。

産業グレードのサーフェス

18 のネイティブ DLL。17 のルーターモジュール。9,500 以上の REST エンドポイント。10,328 のバインド済みエンジン関数。楽しさを生むランタイムの裏側にある、地味な数字たち。

はじめに

1

AI アシスタントを接続

RakuAI MCP サーバーを Claude Desktop（または任意の MCP 対応クライアント）に組み込みます。17 のネイティブツールが会話の中でネイティブに表示されます。Claude Desktop ガイド →

2

ランタイムを駆動

任意の環境で読み取り専用の検査を行うには get_metrics や get_scene_state を呼び出します。サンドボックス／開発環境ではアダプターのロード、フレームの取り込み、レンダーターゲットの設定、シミュレーションの開始ができます。物理・衝突・スコアリングの権威はエンジンが保持します。

3

ビルドログを読む

17 ツールのネイティブサーフェスの設計理由 — なぜこれらのツールなのか、アダプターが次に何を解き放つのか — はブログで解説しています。MCP ビルドログを読む →

17 のネイティブ MCP ツール

外部エージェントはすべて同じ 17 のネイティブツール — get_scene_state、add_scene_node、load_world_model、…（読み取り専用 5 + ミューテーション 12。mcp.html を参照） — を通じてエンジンに到達します。標準コントラクト、任意のエージェント。

対応 AI クライアント

Claude Desktop セットアップガイド →

Claude Desktop

RakuAI MCP サーバーを設定ファイルに追加すれば、17 のネイティブツールが Claude の会話内でネイティブに表示されます。

Cursor

Cursor の MCP 統合経由で、IDE 内から直接ランタイムを駆動。コードを書きながらシーン状態を検査できます。

ChatGPT (OpenAI)

カスタム GPT または Plugin として MCP サーバーを接続。同じ 17 ツールでランタイムを駆動可能。

GitHub Copilot

Copilot Chat の MCP 拡張により、IDE 内から RakuAI ランタイムを呼び出せます。

ランタイムコントラクト

ランタイムは決定論的な権威として動作します。エージェントはツールを呼び出してワールドモデルをロードし、フレームを供給し、状態を読み取ります。物理・衝突・スコアリング・マルチプレイヤー状態はエンジンが管轄します。

完全な MCP コントラクトとスキーマ仕様は 「MCP ツール」解説を参照してください。

API リファレンス

レンダリング、物理、オーディオ、AI、XR、ネットワーキング、アニメーションなどをカバーする包括的な REST API。 MCP サーフェスはこの API レイヤーの上に構築されています。
