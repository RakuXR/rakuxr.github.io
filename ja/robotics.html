ロボティクスの CTO および知覚リード向け

ロボットが平易な言葉で問い合わせる空間メモリ。

あらゆるモバイルロボット — ヒューマノイド、AMR、四足歩行、ドローン — には、コアチームの誰もが作りたがらない 2 つのものが必要です。再起動、シフト交代、ソフトウェア更新をまたいで存続する永続的な空間メモリと、それについて平易な言葉で推論できる LLM です。RakuAI はその両方をサービスとして提供します。一度キャプチャした施設ごとの Gaussian splat を当社がホストし、17 個のネイティブ MCP ツールを介して、お好きな LLM から問い合わせできます。私たちはロボット、SLAM、動作計画は作りません — それらと並んで動きます。

このページは、倉庫自動化、ヒューマノイド、配送、検査ロボティクス企業のエンジニアおよびパートナーシップ担当者に向けて書かれています。私たちがスタックのどこに位置するのか、何を作り何を作らないのか、そして基盤モデルのロボティクス企業が競合ではなく顧客となるのはどこなのかを、正直に伝えることを目指しています。

スタック内での位置づけ。

知覚の上（画像、深度、ポーズを取り込みます）、上位のタスク計画の下（あなたのプランナーの LLM にクエリを提供します）。私たちはシーンメモリ + シーン QA 層です — それ以上でもそれ以下でもありません。その一点集中こそが提案です。

RakuAI が提供するもの

  • 一度キャプチャして永続的にホストする、施設ごとの Gaussian splat
  • あらゆる LLM が呼び出せる 17 個のネイティブ MCP ツール —「7 番通路を塞いでいるのは何？」「火曜日からパレットは動いた？」
  • 再構成されたシーンに対する、平易な言葉でのシーン QA
  • マルチベンダー LLM：Claude、GPT、Gemini、または NVIDIA NIM でホストするオープンウェイト — ワークロードごとにあなたが選択
  • ホスト型の再構成 + クエリインフラ、あなたへのインフラ負担はゼロ

私たちが明確に作らないもの

  • SLAM — Cartographer、RTAB-Map、ORB-SLAM3、NVIDIA cuVSLAM が既に存在し、優れています
  • 動作計画 — MoveIt、OMPL、Nav2 は解決済みの問題です
  • ロボットハードウェアやアクチュエーション — 私たちはソフトウェア専業です
  • シミュレーションエンジン — Isaac Sim、Gazebo、MuJoCo、AWS RoboMaker。私たちは対抗するのではなく、並んで動きます
  • 行動方策の基盤モデル — Skild、Physical Intelligence、Covariant がその層を担います

くさび（差別化点）を一文で：永続的に存続する施設ごとの splat + あらゆる LLM が問い合わせ可能な MCP API — キャプチャ単位・クエリ単位で販売し、ホスト型で、顧客にインフラ負担はありません。堀（モート）は splat ではありません — LLM がベンダーロックされないことです。これは、すでに LLM との関係があり、どれを使うべきか指図されたくないロボット CTO にとって重要です。

ロボットが自分の施設に問い合わせる様子を、1 枚の図で。

ロボットのプランナー（またはその LLM）は、HTTPS 経由で RakuAI の MCP サーフェスと対話します — あるいはロボット搭載の Jetson Orin クライアントからの stdio ブリッジ経由でも構いません。ランタイムは空間プリミティブで応答します。アンカー、ポーズ、シーンクエリ、スプラットハンドル、オブジェクトデルタです。

両方とも実在し、出荷済みです。本番 MCP サーバーは 17 個のネイティブツール（読み取り専用 5 個 + 変更系 12 個）を公開しています — 完全な一覧は engine.html を、ドキュメント化されたシーンツールは mcp.html をご覧ください。ホスト型 MCP は East US の Azure Container Apps 上で稼働しています。LLM はあなたが選べます — そのベンダー中立性こそが堀（モート）です。

このページの対象者。

4 つのセグメント、共通する 1 つの悩み。ロボットは何テラバイトもの空間データを収集しますが、それはストレージに投げ込まれ、二度と問い合わせられることはありません。収益の安定性を狙うなら倉庫から、知名度を狙うならヒューマノイドから始めましょう。

倉庫自動化

ロボットはすでに SLAM を備えています。欠けているのは、カスタムの分析パイプラインを書かずに「前回のシフト以降、ゾーン B で何が変わった？」に答える、すっきりとした手段です。サイトごとのスプラットが、フリート全体の QA を統一します。

想定ターゲット例Symbotic、Locus Robotics、Fetch / Zebra、GreyOrange、Geek+

ヒューマノイドロボット

6〜9 か月で Fortune 500 のパイロットを出荷しようと競う、シリーズ B〜D のスタートアップ。彼らは、特定ベンダーの LLM スタックにロックインされることなく、6 か月分のエンジニアリング項目を取り除けるものなら何でも購入します。

想定ターゲット例Figure、1X Technologies、Apptronik、Agility Robotics、Unitree

配送ロボット

歩道やラストマイルのフリートは、変化し続ける都市環境で稼働します。ルート、ドロップポイント、障害物の永続的なシーンメモリは、ロボットを再計装することなく既存のナビゲーションを補完します。

想定ターゲット例Starship Technologies、Serve Robotics

検査ロボット

Spot や四足歩行のフリートは、膨大なデータセットを生成しますが、それはストレージに投げ込まれ、開かれることはありません。顧客は、監査可能でタイムスタンプ付き、問い合わせ可能なアセットに対してサイトごとに支払います。「第 3 四半期に赤フラグが立ったバルブをすべて見せて」のように。

ROI の計算。

私たちはあなたの知覚スタックを置き換えるわけではありません。誰も担当したがらない永続メモリのサブシステムと、いつまでも「来四半期」へと先送りされるシーン QA の LLM ブリッジを置き換えるのです。

現状 — 自社で構築する場合

  • 知覚エンジニア 1〜2 名が、永続的なシーンメモリの構築に 6〜18 か月を費やします。
  • 総コスト（人件費込み）：エンジニア 1 人年あたり 25 万〜40 万ドル。
  • エンジニア 1 名・6 か月の取り組み：約 12.5 万〜20 万ドル。エンジニア 2 名・18 か月の取り組み：約 75 万〜120 万ドル。
  • その後も永続的に保守が必要 — 当初の労力の年 20〜30% 程度。
  • さらに、メモリを問い合わせ可能な API として公開するための LLM チームとの調整も加わります。

RakuAI を使う場合

  • エンジニア工数ゼロ。
  • キャプチャ単位で支払い：スプラット再構成 1 件あたり 0.50〜2.00 ドルを目標。
  • MCP クエリ単位で支払い：1 クエリあたり数セント（LLM のパススルー + 小幅なマークアップ）。
  • ホスティングは込み。
  • あなたの知覚チームは、より付加価値の高い仕事に取り組めます。

正味の節約、1 年目：25 万〜100 万ドル分のエンジニアリングコストを置き換えます。正直な注意点：見込み顧客がすでに満足のいく永続メモリ + LLM ブリッジのスタックを持っているなら、私たちに売り込む余地はありません — そして、そうでないかのように装うこともしません。

90 日間まるごと無料。成果物は、公開・共同マーケティングするケーススタディです — これが 2 件目、3 件目、4 件目のパイロットを獲得しやすくする納品物になります。
