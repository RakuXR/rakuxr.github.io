RakuAI は、あなたのモデルが空間ワールドを構築・シミュレート・操作する必要があるときに宿るランタイムです。 MCP で接続。カスタム統合は不要。設計段階からマルチベンダー — あなたのモデルは統合の深さではなく品質で競います。

LLM メーカーにとって MCP ネイティブが重要な理由

1. あなたのモデルがそのまま動く。

あなたのアシスタントが MCP を話せば、RakuAI も話せます。6 つのツール。すべてのモデルで同じ契約。カスタム統合は不要。

2. 設計段階からマルチベンダー。

Claude、ChatGPT、Gemini、Copilot — すべて同じやり方で RakuAI を駆動します。あなたのモデルはインテントの品質で競います。

3. 本番はデフォルト拒否。

監査ログ記録。スキーマ検証済み。読み取り専用ツールは本番で動作し、変更ツールはサンドボックス／開発環境で動作します。

アーキテクチャ

何が可能になるか

  • 話す前に観察する NPC — あなたのモデルが毎フレームのシーン状態を読み取る
  • 現実の GPS 座標を中心に再構成される体験
  • シミュレーションステップ上のワールドモデル — あなたのモデルは頭脳であり、コンテンツベンダーではない
  • クロスプラットフォーム配信 — スマートグラス、ヘッドセット、ウェブ、モバイル

Unity との比較

Unity は最近 Native MCP をリリースしました。設計段階のシーン操作には有効です：Claude Desktop がシーンを構築し、Unity がレンダリングします。RakuAI の MCP はシミュレーション時に動作します。あなたのモデルはシーンファイルを生成するのではなく、フレームごとにワールドを駆動します。異なるカテゴリーであり、どちらも有効です。

アダプター契約

6 つのコアツールに加えて、パートナーはアダプターを提供します：

  • VeoEnvironmentAdapter — Google の Veo
  • RunwayEnvironmentAdapter — Runway
  • YourModelAdapter — あなたのモデル

アダプター契約は公開されています。フレーム + シーンデルタを生成できるモデルなら、どれもアダプターになれます。

マルチベンダーの実績

当社の開発ループでは 4 つの AI ベンダーを並行して使用しています：

  • Claude Code がランタイムの大部分を書く
  • Gemini が PR をレビューする
  • ChatGPT がアーキテクチャの壁打ち相手になる
  • GitHub Copilot が他が見逃したものを拾う

当社は特定のモデルに忠実ではありません。当社のランタイムも同じです。
