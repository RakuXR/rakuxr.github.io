Comment ça marche

Votre assistant IA parle MCP. RakuAI est le runtime à l'autre bout de cette connexion, produisant une véritable sortie spatiale vers les lunettes, les casques et le navigateur.

Les 17 outils natifs

La surface MCP native complète : 5 en lecture seule + 12 en mutation. Validée par schéma, journalisée pour audit, contrôlée par permissions. Plus passthrough universel api_call/api_search vers ~9 500 endpoints REST.

Outil Ce qu'il fait Permission
load_world_model(adapter_name, config) Enregistrer un adaptateur de world model Sandbox/dev uniquement
ingest_frame(adapter_name, frame_data, frame_index, timestamp) Injecter une image dans le graphe de scène Sandbox/dev uniquement
get_scene_state(include_physics, include_transforms) Instantané en lecture seule Tous les modes
set_render_target(target_type, config) Configurer la sortie (WebGL/VR/natif/hors écran) Sandbox/dev uniquement
start_simulation(tick_rate, max_duration, realtime) Démarrer la boucle de simulation Sandbox/dev uniquement
get_metrics() Télémétrie de performance Tous les modes

vs Unity MCP

Unity MCP (tout juste publié) : manipulation de scène, placement d'assets, automatisation de build — Claude Desktop génère des scènes statiques au moment de la conception.

RakuAI MCP (en production depuis mars 2026) : orchestration de world model — tout LLM diffuse son intention dans l'étape de simulation à chaque image.

Unity est l'outil que votre IA prend en main. RakuAI est le corps qu'habite votre IA.

Adaptateurs

Le contrat, ce sont les 17 outils natifs. Les adaptateurs sont la façon dont les partenaires se branchent.

Adaptateur personnalisé votre modèle

Le contrat d'adaptateur est public. Construisez-en un, faites partie de l'écosystème.

Pourquoi le MCP-natif compte

  1. Aucune intégration sur mesure — si votre modèle parle MCP, il parle RakuAI.
  2. Multi-fournisseurs par conception — Claude, ChatGPT, Gemini, Copilot pilotent tous les mêmes 17 outils natifs.
  3. Prêt pour la production — refus par défaut, journalisé pour audit, validé par schéma.
