Se connecter EN

Donnez un corps à votre IA dans le monde réel.

RakuAI est le runtime qu'habite votre modèle lorsqu'il a besoin de construire, simuler ou interagir avec un monde spatial. Branchez-vous via MCP. Aucune intégration sur mesure. Multi-fournisseurs par conception — votre modèle se distingue par sa qualité, pas par la profondeur de son intégration.

Se brancher via MCP → Lire l'architecture →

Pourquoi le MCP-natif compte pour les concepteurs de LLM

1. Votre modèle fonctionne, tout simplement.

Si votre assistant parle MCP, il parle RakuAI. Six outils. Le même contrat pour chaque modèle. Aucune intégration sur mesure.

2. Multi-fournisseurs par conception.

Claude, ChatGPT, Gemini, Copilot — tous pilotent RakuAI de la même façon. Votre modèle se distingue par la qualité de l'intention.

3. Refus par défaut en production.

Journalisé pour audit. Validé par schéma. Les outils en lecture seule fonctionnent en production ; les outils de modification en sandbox/dev.

Architecture

Ce qui devient possible

  • Des NPC qui observent avant de parler — votre modèle lit l'état de la scène à chaque image
  • Des expériences qui se reconfigurent autour de coordonnées GPS réelles
  • Des world models au pas de simulation — votre modèle est le cerveau, pas un fournisseur de contenu
  • Livraison multiplateforme — lunettes connectées, casques, web, mobile

vs Unity

Unity vient de publier son MCP natif. C'est bien pour la manipulation de scène au moment de la conception : Claude Desktop construit une scène, Unity la rend. Le MCP de RakuAI s'exécute au moment de la simulation. Votre modèle ne génère pas un fichier de scène — il pilote le monde image par image. Catégorie différente, les deux sont valables.

Contrat d'adaptateur

Au-delà des six outils principaux, les partenaires livrent des adaptateurs :

  • VeoEnvironmentAdapter — Veo de Google
  • RunwayEnvironmentAdapter — Runway
  • YourModelAdapter — votre modèle

Le contrat d'adaptateur est public. Tout modèle qui produit des images + des deltas de scène peut devenir un adaptateur.

Preuves multi-fournisseurs

Notre boucle de développement utilise quatre fournisseurs d'IA en parallèle :

  • Claude Code écrit l'essentiel de notre runtime
  • Gemini relit les PR
  • ChatGPT sert de partenaire pour réfléchir à l'architecture
  • GitHub Copilot repère ce que les autres manquent

Nous ne sommes fidèles à aucun modèle unique. Notre runtime non plus.
