Documentation

Boostez votre IA dans le monde réel.

Pilotez le runtime RakuAI depuis n'importe quel assistant d'IA via la surface MCP native à 17 outils. 18 ms de mouvement-à-photon. Précision spatiale inférieure au millimètre. Conçu pour les lunettes, pas les ordinateurs portables.

Pourquoi RakuAI ?

Cinq brevets américains délivrés. Priorité juillet 2012. Une propriété intellectuelle vieille d'une décennie qui ancre la catégorie actuelle des lunettes AR. Le runtime que Claude, ChatGPT, Gemini et Copilot pilotent tous — multi-fournisseurs par conception.

Conçu pour les lunettes, pas les ordinateurs portables

18 ms de latence mouvement-à-photon. Précision spatiale inférieure au millimètre. Conforme OpenXR sur Meta Quest 3, Xreal Air, Samsung Android XR et MiRZA AR2.

Orchestration de world models

Surface MCP native à 17 outils pour l'orchestration de world models. Chaque image, chaque NPC, chaque interaction routée via l'IA de votre choix. Le MCP d'Unity vise la manipulation de scène au moment de la conception — le MCP de RakuAI s'exécute à l'exécution, à chaque image.

Surface de qualité industrielle

18 DLL natives. 17 modules de routage. Plus de 9 500 endpoints REST. 10 328 fonctions moteur liées. Les chiffres ennuyeux derrière un runtime qui donne envie de jouer.

Démarrage

1

Connectez votre assistant d'IA

Branchez le serveur MCP RakuAI dans Claude Desktop (ou tout client compatible MCP). Les 17 outils natifs apparaissent nativement dans vos conversations. Guide Claude Desktop →

2

Pilotez le runtime

Appelez get_metrics ou get_scene_state pour une inspection en lecture seule dans n'importe quel environnement. En sandbox/dev, chargez des adaptateurs, ingérez des images, configurez des cibles de rendu et lancez des simulations. Le moteur reste l'autorité sur la physique, les collisions et le scoring.

3

Lisez le journal de build

Tout le raisonnement derrière la surface native à 17 outils — pourquoi ces outils, ce que les adaptateurs débloquent ensuite — se trouve dans le blog. Lire le journal de build MCP →

La surface MCP native à 17 outils

Chaque agent externe atteint le moteur via les mêmes 17 outils natifs — get_scene_state, add_scene_node, load_world_model, … (5 en lecture seule + 12 de mutation ; voir mcp.html). Contrat standard, n'importe quel agent.

Clients d'IA pris en charge

Configurer Claude Desktop → pour le script de configuration automatique et la référence complète des outils

Claude Desktop

Prise en charge native du MCP. Script de configuration automatique PowerShell dans le portail développeur ; instructions JSON manuelles pour toute plateforme.

Cursor & clients IDE

Tout éditeur ou IDE compatible MCP peut atteindre le runtime via la même surface native à 17 outils. Guides de configuration à venir.

Agents personnalisés

Créez votre propre agent sur la frontière MCP. Les outils en lecture seule fonctionnent en production ; les outils de mutation sont réservés au sandbox/dev, refusés par défaut.

Apportez votre propre LLM

La surface MCP est agnostique au modèle. Les GPT personnalisés ChatGPT, Gemini, Llama et les modèles internes parlent tous le même contrat.

Vue d'ensemble de l'architecture

Le moteur est l'autorité déterministe. Les world models — prédicteurs vidéo, champs de radiance neuronaux, modèles de fondation de physique — sont des contributeurs qui se branchent via des adaptateurs MCP. Aucun d'eux ne surpasse l'autorité du moteur sur ce qui se passe réellement dans la simulation.

Les schémas d'adaptateurs complets et les kits d'intégration sont disponibles avec l'accès au SDK. Demander l'accès →
