MOTOR ESPACIAL

O Runtime Espacial em C++ que uma LLM Pode Comandar

18 DLLs nativas, 30.738 funções exportadas e um servidor MCP que permite ao Claude, GPT ou qualquer modelo comandar uma cena espacial ao vivo. Não é um substituto do Unity/Unreal — é um runtime de IA espacial com internals transparentes contra os quais você pode fazer link.

Comande uma cena em 10 minutos

O caminho de menor atrito: o servidor MCP contra o Claude Desktop ou o Cursor. Nenhuma instalação do SDK C++ necessária para começar.

1

Clone e configure o servidor MCP

O servidor MCP fica em src/mcp/ no repositório raku-runtime. É um servidor Python — sem necessidade de toolchain C++ para começar a conversar com uma instância em execução.

# configuração única
git clone https://github.com/RakuXR/raku-runtime
cd raku-runtime/src/mcp
pip install -r requirements.txt
2

Aponte o Claude Desktop para o servidor

Adicione a entrada do servidor MCP ao seu claude_desktop_config.json. O Claude Desktop o descobre automaticamente ao reiniciar.

{
  "mcpServers": {
    "raku-runtime": {
      "command": "python",
      "args": ["path/to/raku-runtime/src/mcp/server.py"]
    }
  }
}
3

Peça ao Claude para comandar sua cena

Agora o Claude tem 17 ferramentas MCP nativas e um passthrough para ~9.500 endpoints REST. Crie objetos, consulte o scene graph, execute física — tudo em linguagem natural.

# exemplo de prompt do Claude
"Crie um cubo de corpo rígido na posição (0, 2, 0),
 aplique um impulso para cima de 5 N e liste
 os nós da cena após 1 segundo."
4

Vá mais fundo: faça link direto com uma DLL

Quando o MCP ficar pequeno, faça link diretamente com RakuXR.dll ou RakuRenderer.dll. Todos os 30.738 exports são símbolos públicos — dumpbin /exports mostra todos, sem caixa-preta.

Splat-native: a realidade capturada como objeto de cena

O motor é o substrato do Raku Capture. Um splat gaussiano 3D capturado com a câmera de um celular se torna um objeto de cena de primeira classe — posicionado, ancorado e endereçável por qualquer LLM através das mesmas ferramentas MCP de qualquer outra entidade. O mesmo caminho de runtime roda em XR, desktop e óculos.

Captura → cena em um único pipeline

splat_load faz streaming de um arquivo .spz capturado para a cena ao vivo. O splat herda imediatamente o scene graph, a física e os subsistemas de ancoragem.

Endereçável por LLM desde o primeiro dia

Peça ao Claude para mover um objeto capturado, consultar o que há no splat ou ancorá-lo a uma posição do mundo real — a superfície MCP trata um splat como qualquer outro nó.

Caminho de exibição OpenXR incluído

RakuXR renderiza o splat pelo mesmo pipeline foveado capaz de 120 Hz usado para todo conteúdo de cena — sem visualizador separado.

O pipeline

Escanear → Simular → Renderizar → LLM comanda

Um runtime leva um espaço de uma captura bruta até uma renderização world-locked e fisicamente simulada — o mesmo caminho em XR, desktop e óculos. Uma LLM pode injetar em cada etapa.

01

Escanear

Fusão de sensores + reconstrução constroem uma cena métrica a partir de câmera, profundidade e IMU.

02

Simular

Física determinística, partículas e animação rodam sobre a cena reconstruída.

03

Renderizar

Render foveado capaz de 120 Hz com áudio espacial HRTF, world-locked às âncoras.

04

LLM comanda

17 ferramentas MCP expõem a cena ao vivo a qualquer modelo — criar, consultar, ancorar, alterar.

FÍSICA — RakuPhysics

Motor de Física

Física 3D de nível de produção com corpos rígidos, constraints, raycasting e detecção de colisão. Simulação determinística com tempos de passo de submilissegundo mesmo em hardware de XR móvel. Totalmente endereçável via a ferramenta MCP physics_apply — uma LLM pode aplicar forças, consultar estado de colisão e alterar constraints a partir de um prompt em linguagem natural.

  • Corpos rígidos, juntas, constraints, raycasting
  • Passo determinístico — replays reproduzíveis e simulação conduzida por agente
  • Passo de submilissegundo em hardware de XR móvel
  • 1.278 funções exportadas em RakuPhysics.dll
Benefício-chave: Física consistente e confiável em VR, desktop e mobile — sem ajuste por plataforma. O mesmo passo que roda no headset roda headless em uma avaliação de agente.
VFX — RakuRenderer

Sistema de Partículas e VFX

Sistema de partículas acelerado por GPU com presets de VFX por gênero. Suporta emissores, atratores, colisão e escala de LOD. O gerenciamento automático de orçamento mantém as taxas de quadros travadas em 72–120 Hz em hardware de XR. O renderizador usa DirectX 12 com um back-end Vulkan (RakuVulkan) — ambos os caminhos exportam por RakuRenderer.dll (3.677 exports).

Benefício-chave: VFX cinematográfico que escala com o hardware — sem otimização manual. O mesmo gerenciador de orçamento que governa partículas governa as alterações de cena injetadas por agente.
ANIMAÇÃO — RakuAnimation

Sistema de Animação

Animação esquelética com blend trees, máquinas de estado, IK e retargeting. Suporta personagens animados concorrentes com transições suaves e blending em tempo de execução. A ferramenta MCP animation_play permite que uma LLM dispare ou misture estados de animação diretamente — sem integração personalizada.

  • Blend trees, máquinas de estado, IK, retargeting
  • Blending em tempo de execução — transições conduzidas por LLM
  • 1.900 funções exportadas em RakuAnimation.dll
Benefício-chave: Animação de personagem natural e responsiva sem configuração por personagem. Um agente pode dirigir o comportamento do personagem como um diretor faria — em linguagem natural.
ÁUDIO — RakuAudio

Áudio Espacial

Áudio espacial 3D baseado em HRTF com renderização binaural, oclusão, zonas de reverb e camadas de música adaptativa. Ajustado para alto-falantes de headsets de XR e fones de ouvido com sobrecarga mínima de CPU. A ferramenta MCP audio_play permite que uma LLM posicione sons em qualquer posição 3D na cena.

  • Renderização binaural HRTF — percepção direcional natural
  • Oclusão, zonas de reverb, camadas de música adaptativa
  • 1.654 funções exportadas em RakuAudio.dll
Benefício-chave: Os jogadores localizam sons naturalmente no espaço 3D — crítico para XR imersivo. Âncoras de áudio posicionadas por agente seguem o scene graph como objetos visuais.

Onde o runtime está hoje

Publicamos o estado honesto para que você possa avaliar o runtime com fatos reais, não com marketing.

Fase 0 — Tornando o runtime confiável e consumível

Em produção e confiável: 18 DLLs C++ com 30.738 funções exportadas (verificadas via dumpbin /exports, rastreadas em raku-runtime/dll_exports/ como artefato de CI). Servidor MCP em src/mcp/ com 17 ferramentas nativas ativas. Suporte a OpenXR 1.1 em RakuXR. Membro do NVIDIA Inception (maio de 2026). 5 patentes nos EUA concedidas (prioridade julho de 2012).

Defeito ativo: RakuAssets tem uma correção de linker LNK2001 em andamento. As outras 17 DLLs estão verdes no CI; uma build limpa 18/18 é iminente mas ainda não confirmada. Este aviso será removido quando a correção for aplicada.

Ainda devido (trilhas da Fase 0): documentação de referência por DLL; uma matriz de capacidades de DLL publicada; o vídeo de quickstart "comande sua cena em 10 minutos". Os primeiros embarcadores OEM embutidos são da Fase 1.

Nota sobre a contagem de exports: 30.738 é a baseline documentada/main. Uma build Release local de um branch WIP não mesclado mostra +46 (30.784) — apenas o número 30.738 é canônico para uso público.

Otimizado para XR — e para o envelope térmico dos óculos

Cada subsistema é otimizado para os orçamentos de quadro exigentes de headsets de VR e AR — e para as restrições mais apertadas do hardware de óculos inteligentes, onde Unity e Unreal não miram.

Renderização XR a 120 Hz

Pipeline de renderização foveado com LOD automático mantém os orçamentos de quadro em toda classe de headset.

Inferência SLM On-Device

O RakuSLM mira o envelope térmico abaixo de 5 W do hardware de óculos inteligentes — não rigs de GPU com fio.

Internals Transparentes

Todos os 30.738 exports são símbolos públicos. Faça profiling e otimize no nível de C++ — sem caixa-preta.
