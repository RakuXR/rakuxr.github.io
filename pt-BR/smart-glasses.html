Para líderes técnicos de OEMs de óculos inteligentes

O runtime espacial que os OEMs de óculos inteligentes licenciam—não constroem.

A RakuAI é um runtime espacial neutro quanto a fornecedores de LLM que os OEMs licenciam em vez de construir. Beachhead da Fase 1: Meta Ray-Ban Display PWA — um web app instalável a partir do navegador dos óculos, uma superfície MCP que Claude, ChatGPT, Gemini ou Copilot podem conduzir, e um orçamento térmico/de latência ajustado para displays usados na cabeça. Roadmap da Fase 2: Android XR + SDK nativo (3º tri 2026–2027). Não competimos com o seu hardware — fazemos as experiências de IA nele chegarem mais cedo.

Escrito para líderes de engenharia e parcerias na Meta, Samsung, Google, Xreal, Rokid e outros que lançarão óculos em 2026–2027. Honesto sobre o que já foi lançado, o que é apenas referência e o que depende de hardware. Nenhum parceiro OEM assinado é alegado.

  1. Lendo o material público de lançamento do Meta Ray-Ban Display: o dispositivo suporta web apps instaláveis a partir da superfície web dos óculos. Não assinamos uma parceria com a Meta e não alegamos isso — somos compatíveis com o ponto de entrada público de web app.
  2. O Android XR segue o caminho padrão de instalação de PWA do Chrome. Não assinamos uma parceria com a Samsung ou o Google.
  3. O visionOS atualmente não expõe um caminho de PWA instalável equivalente ao dos óculos. Nosso plano para visionOS é WebXR + um shell nativo separado como continuação.
  4. “Captura a partir dos óculos hoje” significa: o usuário veste o Ray-Ban Display, o celular companheiro roda nosso PWA e o celular é a fonte-de-verdade da câmera. A captura direta de frames dos óculos chega quando o OEM expuser a API de frame.

Três fases. Um modelo de licenciamento.

O go-to-market é faseado pela prontidão da plataforma — não pelo cronograma da RakuAI. Encontramos o seu dispositivo nas superfícies públicas que ele já expõe e, depois, aprofundamos a integração quando você estiver pronto.

Fase 1 — Beachhead

Meta Ray-Ban Display PWA · meta 3º tri 2026

  • Entrada: PWA instalável a partir do navegador embarcado dos óculos — sem gating de app-store, sem instalar SDK no seu dispositivo
  • Superfície MCP: 17 ferramentas nativas + 6 ferramentas universais de passthrough, ativas para Claude, ChatGPT, Gemini, Copilot
  • Captura: celular-companheiro-como-câmera hoje; captura direta dos óculos quando a API de frame do OEM chegar (dependente de hardware)
  • Arquitetura: celular-como-compute (o celular roda IA/física, os óculos exibem + rastreiam)
  • Rótulo honesto: o caminho PWA está pronto como referência; nenhum co-marketing ou pré-instalação com a Meta está acordado

Fase 2 — Plataforma

Android XR + SDK nativo · 2027

  • Entrada: o mesmo caminho PWA se generaliza pela superfície Chrome padronizada do Android XR
  • SDK nativo: o raku-runtime como DLLs nativas com uma camada de sessão compatível com OpenXR — integrado ao compositor, ao pipeline de rastreamento de mãos ou ao driver de display do OEM
  • Meta de docs do SDK: docs de nível parceiro no 4º tri 2026; caminho de integração de 8–12 semanas-dev
  • Parceiros: Samsung Galaxy XR + licenciados Android XR; Xreal, Rokid, navegador do Vision Pro na mesma oferta faseada
  • Rótulo honesto: janela de lançamento do Android XR sujeita ao cronograma da plataforma; nenhum OEM assinado

Fase 3 — Fan-out

Xreal / Rokid / Galaxy XR / Vision Pro · 2027+

  • Suporte a dispositivos de cauda longa pelo caminho do navegador
  • Resiliência offline / de link intermitente para óculos de menor especificação
  • Pipeline repetível autoqualificar → piloto PWA → SDK
  • App de lançamento com co-marketing para um parceiro estratégico Tier 3
  • Apresentações calorosas do NVIDIA Inception ao BD do ecossistema OEM de XR

Rede NVIDIA Inception: a RakuAI foi aceita no NVIDIA Inception (maio de 2026). Essa rede oferece apresentações calorosas ao ecossistema de hardware XR — muitos OEMs de óculos e seus fornecedores de silício/óptica são membros do Inception ou parceiros da NVIDIA.

Integração OEM, passo a passo.

Deslize pelo caminho da primeira conversa até uma experiência co-lançada. Sem lock-in de SDK — encontramos o seu dispositivo nas superfícies web públicas que ele já expõe.

1

Conecte pelo PWA

A RakuAI instala como um PWA a partir da superfície web do seu dispositivo — sem gating de app-store. Seus óculos carregam rakuai.com/capture-app/ e o runtime fica ativo.

  • Entrada de web app instalável, hoje
  • Caminho de cliente WebXR / WebGPU
  • Nenhum shell nativo necessário para começar
2

Conduza com qualquer LLM via MCP

Claude, ChatGPT, Gemini ou Copilot conduzem o runtime pelas 17 ferramentas MCP nativas — via uma ponte stdio ao dispositivo companheiro ou por HTTPS ao MCP hospedado.

  • 17 ferramentas MCP nativas (5 de leitura + 12 de mutação)
  • 6 ferramentas adaptadoras xr_ para fluxos de óculos
  • Neutro quanto a fornecedores — quem usa escolhe o LLM
3

Capture o espaço de quem usa

Frames + IMU fluem para a reconstrução em nuvem e voltam como um splat gaussiano explorável em minutos — do celular companheiro hoje, direto dos óculos quando a sua API de câmera chegar (dependente de hardware: requer a API de fonte de frame do OEM).

  • Reconstrução GLOMAP + Brush, saída .spz
  • Captura por celular companheiro já disponível
  • Direto dos óculos no caminho do 3º tri 2026
4

Caiba no envelope térmico

Celular-como-compute mantém os óculos frios: o celular roda física e IA, os óculos renderizam e rastreiam, o link carrega deltas de cena — não vídeo. Três modos térmicos se adaptam ao dispositivo.

  • Meta de 18 ms motion-to-photon
  • Sync de delta de cena ~20 Hz, 400–1400 B
  • Modos Performance / Balanceado / Baixo Consumo
5

Co-lance sob NDA

Acesso sob NDA mútuo às ferramentas adaptadoras xr_, à prévia de captura a partir dos óculos e ao pipeline de splat WebGPU — com um caminho para uma experiência de lançamento co-marketada para o seu dispositivo.

  • Prévia do SDK OEM sob NDA
  • Licenciamento de patentes em nível de arquitetura disponível
  • NDA mútuo em dias, não semanas

MCP a partir dos óculos, em um diagrama.

Um LLM nos óculos conversa com a superfície MCP da RakuAI pelo PWA — seja via uma ponte stdio ao dispositivo companheiro ou por HTTPS ao MCP hospedado. O runtime responde em primitivas espaciais: âncoras, poses, consultas de cena, handles de splat.

As duas metades são reais e já entregues. O servidor MCP de produção expõe 17 ferramentas nativas (5 de leitura + 12 de mutação) — veja spatial-engine.html para o inventário completo. As ~12 relevantes para óculos se dividem em: 6 ferramentas principais de captura/cena (documentadas na página MCP) e 6 ferramentas adaptadoras xr_ (sessão, pose da cabeça, entrada e primitivas de âncora para fluxos conduzidos pelos óculos). O MCP hospedado roda no Azure Container Apps em East US.

O patrimônio de PI por trás do runtime.

Cinco patentes dos EUA concedidas, prioridade julho de 2012, patrimônio até julho de 2032. A arquitetura que esta página descreve é a arquitetura que essas patentes previram.

  • US9781244B2 Óculos Inteligentes (concedida out 2017)
  • US9210358B2 Console de Entretenimento (concedida dez 2015)
  • US9654613B2 Arena Gaming (concedida mai 2017)
  • US10027361B2 Point-of-Service (concedida jul 2018)
  • US10432772B2 Óculos (21 reivindicações)

O que as reivindicações cobrem

  • Comunicação sem fio de modo duplo (baixa latência + alta largura de banda)
  • Arquitetura de óculos inteligentes + host de compute/celular
  • Offload de render do host de compute para os óculos
  • Sincronização de estado de mundo multiplayer
  • Streaming de mundo (entrega de delta de cena, não vídeo)

Essas patentes antecedem os Meta Ray-Bans (2021), o Xreal Air (2022), o Meta Ray-Ban Display (2024–25) e o Android XR (2026). Para OEMs que querem um caminho claro adiante em reivindicações de nível de arquitetura, oferecemos licenciamento ao lado da parceria.

Arquitetura celular-como-compute.

A divisão que as patentes de 2012 previram — agora a única arquitetura que cabe no envelope térmico dos óculos.

O celular roda o mundo

Física (60 Hz fixos), comportamento de IA, estado do jogo, streaming de assets.

Os óculos exibem o mundo

Predição de pose local, renderização foveada, IMU + odometria visual.

O link faz o resto

Delta de cena sobre Wi-Fi 7 (com fallback óptico quando disponível), uplink de sensores, sem necessidade de streaming de vídeo.
