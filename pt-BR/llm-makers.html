Arquitetura

O que é possível

  • NPCs que observam antes de falar — seu modelo lê o estado da cena a cada quadro
  • Experiências que se reorganizam em torno de coordenadas GPS reais
  • World models no passo de simulação — seu modelo é o cérebro, não um fornecedor de conteúdo
  • Entrega multiplataforma — smart glasses, headsets, web, mobile

vs Unity

O Unity acaba de lançar o Native MCP. Ele é bom para manipulação de cena em tempo de design: o Claude Desktop monta uma cena, o Unity a renderiza. O MCP da RakuAI roda em tempo de simulação. Seu modelo não está gerando um arquivo de cena — ele está dirigindo o mundo quadro a quadro. Categoria diferente, ambas válidas.

Contrato de adaptador

Além das seis ferramentas centrais, os parceiros entregam adaptadores:

  • VeoEnvironmentAdapter — Veo do Google
  • RunwayEnvironmentAdapter — Runway
  • YourModelAdapter — seu modelo

O contrato de adaptador é público. Qualquer modelo que produza quadros + deltas de cena pode se tornar um adaptador.

Comprovações multifornecedor

Nosso loop de desenvolvimento usa quatro fornecedores de IA em paralelo:

  • O Claude Code escreve a maior parte do nosso runtime
  • O Gemini revisa os PRs
  • O ChatGPT serve de "pato de borracha" para a arquitetura
  • O GitHub Copilot pega o que os outros deixam passar

Não somos fiéis a nenhum modelo específico. Nosso runtime também não.
