Why MCP-native matters for LLM makers

1. Connect once, drive everything.

If your assistant speaks MCP, it speaks RakuAI. One JSON-Schema contract, versioned and stable: 17 native tools (5 read + 12 mutation) plus ~147 passthrough endpoints drawn from ~9,500 available via api.rakuai.com. No custom integration, no per-vendor fork.

2. Multi-vendor by design.

The contract is identical for Claude, ChatGPT, Gemini, and Copilot — identical by construction. No exclusivity. Per-vendor end-to-end conformance testing is in progress; we will publish results before claiming verified parity for any vendor.

3. Inference-only. We don't train.

RakuAI is a spatial runtime your model calls at inference time. We hold no model weights, run no training jobs against your data, and fine-tune nothing. Your data stays yours. This is a hard line.

4. Deny-by-default in production.

Read tools (get_scene_state, list_objects, get_metrics, get_world_model, get_render_target) are live in production. Mutation tools (set_render_target, start_simulation, and 10 others) are sandboxed and audit-logged by default.

Architecture

What's possible

  • NPCs that observe before they speak — your model reads scene state every frame (available today)
  • World models on the simulation step — your model is the brain, not a content vendor (available today)
  • Cross-platform delivery — smart glasses, headsets, web, mobile (available today)
  • Experiences that re-form around real GPS coordinates (roadmap)

vs Unity

Unity just released Native MCP. It's good for design-time scene manipulation: Claude Desktop builds a scene, Unity renders it. RakuAI's MCP runs at simulation time. Your model isn't generating a scene file — it's driving the world frame by frame. Different category, both valid.

Adapter contract (roadmap)

The planned extension surface: partners ship adapters that translate model-specific outputs into the RakuAI scene contract. The adapter interface is being designed now; no live adapters exist yet.

  • VeoEnvironmentAdapter — planned: Google's Veo (roadmap)
  • RunwayEnvironmentAdapter — planned: Runway (roadmap)
  • YourModelAdapter — your model, once the contract ships

Adapter contract design is in progress. Any model that produces frames + scene deltas will be able to become an adapter. We will publish the spec publicly before requesting implementations.
