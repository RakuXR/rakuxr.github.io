從您的 AI 助理驅動 RakuAI。

17 個原生 MCP 工具（5 個唯讀 + 12 個變更）加通用 api_call/api_search 透傳至約 9,500 個 REST 端點。自 2026 年 3 月起已部署於生產環境——現具版本化公開工具合約、5 分鐘唯讀連線路徑，以及預設拒絕、稽核日誌式的權限機制。

運作方式

您的 AI 助理會說 MCP。RakuAI 是這條連線另一端的執行階段，向眼鏡、頭戴裝置與瀏覽器驅動真實的空間輸出。

17 個原生工具

完整的原生 MCP 介面：5 個唯讀 + 12 個變更工具。結構驗證、稽核日誌、權限控管。加通用 api_call/api_search 透傳至約 9,500 個 raku-api REST 端點（完整清單見 spatial-engine.html）。17 工具介面現已釘選為版本化公開合約（tools.v1.json，schema_version 1），並設有反偏移測試，若公開介面與即時登錄不符即令 CI 失敗。

5 分鐘連線——唯讀

只要您的模型會說 MCP，它就會說 RakuAI。將現有 MCP 用戶端指向執行階段，即可在免費沙箱上取得可用的唯讀工作階段——5 個唯讀工具——無需綁定特定模型、無需引擎 SDK、無變更風險。

新增伺服器 在您的 MCP 用戶端設定（Claude Desktop 或任何支援 MCP 的助理）中登錄 raku-mcp。
啟動唯讀工作階段 唯讀模式為預設。5 個唯讀工具在所有模式下皆可運作，包含生產環境。
讀取即時場景狀態 呼叫 get_scene_state 和 get_metrics，觀察執行階段回應。這立即帶來實用價值。
升級至沙箱 12 個變更工具在沙箱／開發模式下解鎖，讓您端到端驅動場景。

逐步說明：唯讀連線快速入門、LLM 整合指南，以及驅動場景逐步說明。站內：Claude Desktop 指南。

多廠商平等性

無論呼叫方是 Claude、ChatGPT、Gemini 或 Copilot，合約完全相同——這正是我們的優勢所在。我們發布一份平等性矩陣，追蹤每個廠商對全部 17 個原生工具加透傳的支援情況。平等性在協定層面成立；特定用戶端的 schema 處理方式作為持續更新的公開記錄被明確追蹤。

稽核與權限說明

讓安全負責人批准在空間中自主行動的代理，需要什麼條件？變更操作在設計上受到管控，每次呼叫皆被記錄，且中繼層強制執行分級限制。

預設拒絕 全部 12 個變更工具預設為沙箱／開發模式；只有 5 個唯讀工具在所有模式下皆可運作。生產環境的變更操作需要明確且經過稽核的權限授予。
每工作階段稽核日誌匯出 每次中繼呼叫皆被記錄；帳戶擁有者可跨 /api/raku/* 介面匯出其每工作階段稽核追蹤，支援安全遮罩。
速率限制與配額 免費與付費方案在中繼介面上具備速率限制與配額機制，確保使用量有界且按方案計責。

誠實說明：稽核匯出今日為中繼範圍且跨帳戶，尚未達到每連接器粒度；配額強制在中繼計數器存儲已設定的情況下啟用。每連接器粒度與簽署稽核日誌列於路線圖。

對比 Unity MCP

Unity MCP（剛發布）：場景操作、資產擺放、建置自動化——Claude Desktop 在設計階段生成靜態場景。

RakuAI MCP（自 2026 年 3 月起進入生產）：世界模型編排——任何 LLM 在每一幀將意圖串流進模擬步進。

Unity 是您的 AI 拿起來用的工具。RakuAI 則是您的 AI 棲身的身體。

為什麼 MCP 原生很重要

  1. 無需客製化整合——只要您的模型會說 MCP，它就會說 RakuAI。以唯讀模式連線約需五分鐘。
  2. 設計上即多廠商——Claude、ChatGPT、Gemini 與 Copilot 皆針對相同的 17 個原生工具；平等性在協定層面成立。
  3. 版本化合約——17 工具介面已釘選為 tools.v1.json 並設有反偏移測試，確保您整合的介面不會悄悄改變。
  4. 可防守的權限——預設拒絕、稽核日誌、結構驗證，並具備每工作階段稽核日誌匯出供安全審查。
