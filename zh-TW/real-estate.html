為房仲、仲介公司與物件團隊而寫

讓任何物件成為可探索的 3D 空間。

用手機掃描一處物件 — 不需要 300 美元的設備、不需要特殊相機 — RakuAI 就會將其重建為可探索的 3D 高斯潑濺導覽。接著，買家便能做到影片導覽做不到的事：向物件提問 —「廚房中島有多寬？」、「主臥朝哪個方向？」— 由你或你的入口網站已在使用的任一 LLM 透過我們的 MCP 介面回答。

這是 RakuAI 的旗艦垂直領域，而我們正以公開的方式打造它。下方頁面誠實說明今日已交付的部分與藍圖上的部分。我們尚未營收，正在尋找搶先體驗的房仲與試行的仲介公司 — 並非宣稱已是完成的產品。

申請搶先體驗 → 了解運作方式 ↓

從手機走訪，到買家可以對話的物件。

驅動平台其餘部分的同一條 Raku Capture 管線 — 一次一個步驟。使用方向鍵、點點或鍵盤操作。

步驟 1

用手機掃描

拿著口袋裡早就有的手機走訪物件。不需要三腳架、不需要 300 美元的擷取設備、不需要特殊相機，也不需要訓練日。

步驟 2

重建為 3D 潑濺

影格與動作在雲端重建為一個 3D 高斯潑濺 — 一個平滑、趨近照片寫實的空間模型，而非拼接的全景。

步驟 3

買家在瀏覽器中探索

分享一條連結。買家以自己的步調、用任何裝置穿梭空間 — 不需要頭戴裝置、不需要安裝 App、不需要預約看屋。

步驟 4

向物件提出任何問題

透過我們的 MCP 介面，LLM 以實際掃描為依據回答問題 — 尺寸、朝向、房間裡有什麼。多廠商：Claude、ChatGPT、Gemini 或 Copilot 皆可驅動。

運作方式。

從手中的手機到一份可分享、可提問的物件，只需三個步驟。這就是貫穿 RakuAI 各處所使用的同一個 Raku Capture 核心 — 不動產只是套在輸出之上的一層包裝，而非全新引擎。

1

掃描

拿著手機走訪住家，螢幕上有引導。每個空間擷取只需數分鐘，所需的東西你早就有了。

2

重建並發佈

RakuAI 將掃描重建為可探索的 3D 高斯潑濺，並提供你一條可放入物件頁面、電子郵件或入口網站的單一分享連結。

3

探索並提問

買家在瀏覽器中走訪空間，並以白話提問 — 由任何 LLM 透過 MCP 介面、以掃描為依據回答。

這與 Matterport 導覽有何不同。

3D 物件導覽並不新鮮。新鮮的是用手機完成它、讓空間回答問題，並且永不把你綁定在單一廠商。

典型 3D 導覽（例如 Matterport） 面向不動產的 RakuAI
擷取硬體 通常需要專用相機或約 300 美元以上的設備，或一次付費的掃描技師到府 你早就擁有的手機 — 無需特殊相機
重建 全景／網格走訪 3D 高斯潑濺 — 更平滑、更趨近照片寫實的導覽
向空間提問 並非原生支援 — 它是一個檢視器，而非問答介面 可以 — 可透過 MCP 由 LLM 查詢，以掃描為依據
由哪個 AI 回答 不適用，或單一廠商 多廠商 — Claude、ChatGPT、Gemini 或 Copilot，由你選擇
廠商綁定 綁定於單一平台與檢視器 中立、基於標準的 MCP 介面；自帶你的模型

此對照反映的是公開且典型的工作流程，並非基準測試；其他產品的能力與定價會有所變動。我們指出正在朝其建構的差異，並在下方誠實說明尚未完成的部分。

為何房仲與仲介公司在意。

這個賣點不是「更花俏的導覽」。它是更少的無謂看屋、能預先篩選買家的物件，以及低到可以套用在每一個物件上的擷取成本 — 而不只是豪宅。

每一個物件，不只是豪宅

當擷取是一次手機走訪，而不是付費的掃描技師到府時，沉浸式導覽就不再是只為高端住宅保留的成本項目。邊際成本趨近於零，因此可以套用到整本物件冊。

更少低意願的看屋

能探索空間並先得到基本問題解答的買家，往往會自我篩選。你實際安排的實體看屋，對象都是已經在虛擬走訪過的人。

凌晨兩點也能回答的物件

「車庫放得下兩台車嗎？」「臥室朝哪個方向？」物件本身就以掃描為依據回答這些問題，無論買家是何時在瀏覽 — 而不只是辦公時間。

你的 AI、你的入口網站

由於它是中立的 MCP 介面，仲介公司或入口網站可以將物件的問答接入它已在付費使用的任一 LLM，並以自己的品牌呈現 — 無需被迫更換廠商。
