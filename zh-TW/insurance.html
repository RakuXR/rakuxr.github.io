為屋主、租客及其保險公司而寫

一次手機掃描，即可生成保險公司可用的物品清單。

RakuAI Capture 將一次 60–90 秒的房間手機掃描，轉化為一份逐項列出室內物品的清單 — 附帶概略尺寸與預估重置價值 — 你可直接交給保險公司。專為人們真正需要它的三個時刻而打造：申請理賠、規劃搬家，以及縮減家當。採用與 RakuAI 其餘部分相同的 3D Gaussian splat 重建與 MCP 核心 — 並非全新引擎。

本頁誠實說明哪些已於今日交付、哪些仍列於藍圖。我們記錄一個空間並產出一份結構化清單；我們並非保險公司，不訂定保單條款，也不和解理賠。

人們需要它的三個時刻。

我們刻意把不動產上架擷取讓給既有業者（Matterport 及同類）。這是無人佔據的消費者切入點：一次自助掃描，在清單真正派上用場的時刻，產出一份結構化的物品清單。

理賠

發生損失之後 — 火災、竊盜、水損、風災 — 憑記憶重建自己擁有過什麼既痛苦又容易引發爭議。在投保或續保時做一次掃描，便能在任何事發生前，留下一份附時間戳記、逐項列出的紀錄。申請理賠從「回想你損失的每一樣東西」變成「提交清單」。

搬家

搬家公司、估價保障與運送保險都需要一份物品清單。每個房間一次掃描，即可產出附粗略尺寸的逐項清單 — 對搬家報價、以及記錄裝車前後的狀況都很有用。

縮減家當

清空一個家 — 處理遺產、父母搬入安養機構、孩子離家上大學 — 意味著得決定哪些要留、要賣、要捐、要投保。一次掃描讓全家人對房間裡有什麼，共享一份逐項列出的同一視圖。

一次掃描如何變成一份清單。

這就是貫穿產品其餘部分所使用的同一條 Raku Capture 管線 — 掃描、重建、查詢。保險只是套在輸出之外的一層包裝，而非全新引擎。

1

掃描房間

拿著手機緩慢繞行一圈，60–90 秒，螢幕上有引導。無需特殊硬體。

2

重建

影格 + 動作在雲端重建為一個 3D Gaussian splat — 一個可探索的空間模型。

3

逐項列出

LLM 透過 MCP 介面巡遍場景，列出可見物件，並為每件物品附上概略尺寸。

4

檢視與匯出

你編輯清單、確認價值，並匯出一份可交給保險公司的乾淨清單。每項價值皆標示為預估值。

輸出長什麼樣子。

一份示意用的單一房間清單。實際輸出由你的掃描生成，且在你提交給任何人之前皆可完整編輯。

我們寧可保守承諾。以下是目前已上線與藍圖之間的分界。

今日已交付

  • 以手機進行的擷取與雲端 splat 重建（Raku Capture 流程）。
  • 透過 MCP 介面的場景查詢 — LLM 可巡遍已重建的房間並描述其中有什麼。
  • 多 LLM 支援：Claude、ChatGPT、Gemini 與 Copilot 皆可驅動執行階段。

進行中／藍圖

  • 一份精緻的單頁保險公司可用匯出（房間縮圖 + 逐項表格 + 總計 + 掃描中繼資料）。底層資料今日已存在；格式化的文件正在建構中。
  • 各保險公司專屬的清單範本 — 由於沒有單一的業界標準格式，以逐案試行的方式處理。
  • 按掃描次數計費，以及一個消費者變更追蹤方案（每年重新掃描，查看差異）。

我們尚未具備的

  • SOC 2 認證。大型保險公司在採購前要求此項；我們在這項就緒工作並行推進的同時，先與 InsurTech 進行早期試行。
  • 任何已簽署的保險公司合作關係。我們不主張獲得任何保險公司背書。
  • 鑑價權限。我們記錄並估算；我們不鑑價，也不和解。
