為什麼 MCP 原生對 LLM 製造商很重要

1. 您的模型直接就能用。

只要您的助理會說 MCP，它就會說 RakuAI。六個工具。每個模型都套用相同契約。無需客製化整合。

2. 設計上即多廠商。

Claude、ChatGPT、Gemini、Copilot — 它們全都以相同方式驅動 RakuAI。您的模型憑意圖品質競爭。

3. 生產級預設拒絕。

稽核紀錄。結構驗證。唯讀工具可在生產環境運作；變更工具則限於沙箱／開發環境。

架構

能實現什麼

  • 先觀察再開口的 NPC — 您的模型在每一個影格讀取場景狀態
  • 圍繞真實 GPS 座標重新成形的體驗
  • 在模擬步進上運行的世界模型 — 您的模型是大腦，而非內容供應商
  • 跨平台交付 — 智慧眼鏡、頭戴裝置、網頁、行動裝置

對比 Unity

Unity 剛發布了原生 MCP。它擅長設計階段的場景操作：Claude Desktop 建構場景，Unity 渲染它。RakuAI 的 MCP 則在模擬時運行。您的模型不是在產生場景檔案 — 而是逐影格驅動世界。不同的類別，兩者都成立。

轉接器契約

除了六個核心工具之外，合作夥伴還可出貨轉接器：

  • VeoEnvironmentAdapter — Google 的 Veo
  • RunwayEnvironmentAdapter — Runway
  • YourModelAdapter — 您的模型

轉接器契約是公開的。任何能產生影格 + 場景差異的模型都能成為一個轉接器。

多廠商的真憑實據

我們的開發循環同時並用四家 AI 廠商：

  • Claude Code 撰寫我們執行階段的大部分
  • Gemini 審查 PR
  • ChatGPT 充當架構討論的「橡皮鴨」
  • GitHub Copilot 抓出其他工具漏掉的部分

我們不效忠任何單一模型。我們的執行階段也不會。
