為何選擇 RakuAI
US 5 項美國已獲核准專利 • 優先權日：2012 年 7 月

在現實世界中為您的 AI 加速

RakuAI 是 AI 原生執行階段，透過 MCP 讓 Claude、ChatGPT、Gemini、Copilot 變成空間代理。 六個工具。預設拒絕。稽核紀錄。生產部署。為產業正要出貨的智慧眼鏡而生。

Unity MCP：設計階段的場景操作。RakuAI MCP：執行階段的世界模型編排，每一個影格。
Unreal 需要 RTX 4090；RakuAI 只需要 Wi-Fi 7 與一支手機。

了解 API 親眼看引擎運作

其他引擎給不了的三件事

Unity、Unreal 與 Godot 的架構誕生於 ChatGPT 之前、Quest Pro 之前，也在 WebGPU 之前。 RakuAI 則從第一天就為這三者而設計。

☷
單一程式碼庫

到處用同一套 API

從瀏覽器、Windows 原生用戶端，或 XR 頭戴裝置呼叫同一組 REST 端點。 沒有獨立的 WebGL 移植版、沒有功能精簡的行動版、沒有特定平台的怪異陷阱。 建置一次，同步出貨到 Web、原生與沉浸式裝置。

☻
代理優先

AI 原生引擎

LLM 驅動的 NPC、裝置端小型語言模型推論、情緒建模與對話樹皆為一級子系統 — 不是事後才硬塞進去的外掛。引擎一開始就為了能看、能想、能說的代理而設計。

⚙
生產規模

10,000+ 個函式

18 個原生子系統。10,328 個引擎函式。9,500+ 個 REST 端點。 物理、渲染、XR、音訊、動畫、網路、腳本 — 真實引擎完整的表面積， 不是框架或示範。經實戰驗證，可承擔生產級工作負載。

不是 Unity。 不是 Unreal。 不是 Godot。
全新的東西。

既有的引擎是為原生二進位、單機桌面遊戲的世界而打造。 RakuAI 則為每個角色都會說話、每個頭戴裝置出廠就支援手部追蹤、每個遊戲都離一個瀏覽器分頁不遠的世界而打造。 地基本來就不該一樣。
