企業空間 AI

一次性空間擷取。任何 LLM 都能呼叫的 MCP 執行階段 — 17 個原生工具、約 147 個透傳工具、約 9,500 個 API 端點。為企業垂直整合而打造：保險（理賠與物品清單）、零售與設施（空間規劃、變更偵測）以及機器人（持久空間記憶）— 並誠實標註哪些已正式發布（GA），哪些在藍圖中。

企業垂直整合

三個垂直領域，一套空間擷取 + MCP 基底。保險與機器人是已獲資金支持的活躍切入點。零售與設施以需求驅動方式服務 — 當設計合作夥伴確定後即升級為獲資金支持的切入點。

三者均執行於同一 Raku Capture 流水線 + MCP 工具介面上： api.rakuai.com.

大規模部署空間 AI

滑動瀏覽企業部署的樣貌 — 在單一執行階段上的擴展性、安全與治理。

部署

邊緣 + 執行階段裝置將資料串流至 RakuAI 執行階段；控制平面與資料路徑保持分離。
雙裝置手機作運算、眼鏡作顯示 — 原生設計，而非外掛拼接。
OpenXR基於標準的算繪，實現廣泛的頭戴裝置相容性。

擴展

全叢集政策一套政策適用於全域，而非逐台裝置設定。
18ms 目標在所有受支援硬體上維持動到光延遲目標。
熱感知動態品質回退使裝置保持在其運作範圍內。

安全與治理

預設拒絕除非政策明確允許，否則任何操作都不會執行。
RBAC + SSO/SCIM基於角色的存取控制，整合企業身分系統。
不可變稽核記錄每一項決策，用於合規與事件審查。

治理與身分  Roadmap

企業採購所需的認證、稽核與叢集政策層。處於第 1 階段建構中 — 尚未投入生產。

  • SSO (OIDC / SAML) — 第 1 階段
  • SCIM 佈建 — 第 1 階段
  • 預設拒絕 RBAC — 第 1 階段
  • AI 操作稽核日誌 — 第 1 階段
  • SOC 2 就緒評估 — 進行中

自訂 MCP 與 OEM  Roadmap

依租戶的自訂 MCP 命名空間（你的系統可與我們的系統一起被呼叫）及私有 / 白標端點。第 2 階段目標。

  • 依租戶的 MCP 工具命名空間 — 第 2 階段
  • 私有 / 品牌 MCP 端點 — 第 2 階段
  • 資料落地選項 — 第 3 階段
  • 自訂品牌 / OEM 套件 — 第 2 階段
  • 合約 SLA — 第 2 階段

第 1 階段 = 正在建構。第 2–3 階段 = 已獲資金的藍圖，尚未 GA。暫無 SOC 2 報告；就緒評估進行中。

誠實現況： 擷取 + 查詢已投入生產。SSO/SCIM、預設拒絕 RBAC、稽核日誌、自訂 MCP 命名空間、私有/品牌端點、資料落地及 OEM 打包均為尚未 GA 的藍圖項目。今天要求 SOC 2 的採購方將得到誠實的「就緒評估進行中」答覆。沒有虛構客戶，沒有尚無值班支援的 SLA。當你聯絡我們時，我們會準確告知有哪些功能可用。
