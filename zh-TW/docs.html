在現實世界中為您的 AI 加速。

從任何 AI 助理透過 17 個原生 MCP 工具介面驅動 RakuAI 執行階段。18ms 動到光延遲。亞毫米級空間精度。為眼鏡而生，不是為筆電。

為何選擇 RakuAI？

5 項美國已獲核准專利。優先權日 2012 年 7 月。歷時十年、為今日 AR 眼鏡品類奠定基礎的智慧財產。Claude、ChatGPT、Gemini、Copilot 全都能驅動的執行階段 — 多廠商設計。

為眼鏡而生，不是為筆電

18ms 動到光延遲。亞毫米級空間精度。在 Meta Quest 3、Xreal Air、Samsung Android XR 與 MiRZA AR2 上皆符合 OpenXR 合規。

世界模型編排

用於世界模型編排的 17 個原生 MCP 工具介面。每一個影格、每一個 NPC、每一次互動都透過您選擇的 AI 進行路由。Unity 的 MCP 鎖定設計階段的場景操作 — RakuAI 的 MCP 則在執行階段運作，每一個影格。

工業級介面

18 個原生 DLL。17 個路由模組。9,500+ 個 REST 端點。10,328 個綁定的引擎函式。撐起這個玩起來有趣的執行階段背後的枯燥數字。

立即開始

1

連接您的 AI 助理

將 RakuAI MCP 伺服器接入 Claude Desktop（或任何支援 MCP 的用戶端）。這 17 個原生工具會原生出現在您的對話中。Claude Desktop 指南 →

2

驅動執行階段

在任何環境中呼叫 get_metrics 或 get_scene_state 進行唯讀檢視。在 sandbox/dev 中，可載入轉接器、擷取影格、設定渲染目標並啟動模擬。引擎仍掌握物理、碰撞與計分的權威。

3

閱讀開發記錄

17 個原生 MCP 工具介面

每個外部代理都透過同樣的 17 個原生工具存取引擎 — get_scene_state、add_scene_node、load_world_model、…（5 個唯讀 + 12 個變更；詳見 mcp.html）。標準合約，適用任何代理。

支援的 AI 用戶端

串接 Claude Desktop → 以取得自動設定指令碼與完整的工具參考

Claude Desktop

原生 MCP 支援。開發者入口提供 PowerShell 自動設定指令碼；亦提供適用任何平台的手動 JSON 說明。

Cursor 與 IDE 用戶端

任何支援 MCP 的編輯器或 IDE 都能透過同一套 17 個原生工具介面存取執行階段。串接指南即將推出。

自訂代理

在 MCP 邊界上建構您自己的代理。唯讀工具可在生產環境運作；變更類工具僅限 sandbox/dev，且預設拒絕。

自帶 LLM

MCP 介面與模型無關。ChatGPT 自訂 GPT、Gemini、Llama 與內部模型都能說同一套合約。

架構總覽

引擎是確定性的權威。世界模型 — 影片預測器、神經輻射場、物理基礎模型 — 是透過 MCP 轉接器接入的貢獻者。它們都不會凌駕引擎對於模擬中實際發生之事的權威。
