三個階段。一種授權模式。

市場進入按平台就緒度分階段 — 而非按 RakuAI 的時間表。我們在你的裝置已暴露的公開表面上與之相遇，待你準備好後再深化整合。

第 1 階段 — 灘頭陣地

Meta Ray-Ban Display PWA · 目標 2026 年第三季

  • 入口：從眼鏡的嵌入式瀏覽器安裝的 PWA — 無應用商店門檻，無需在你的裝置上安裝 SDK
  • MCP 表面：17 個原生工具 + 6 個通用透傳工具，對 Claude·ChatGPT·Gemini·Copilot 即時可用
  • 擷取：目前以配套手機作為相機；OEM 影格 API 落地後從眼鏡直接擷取 (受硬體限制)
  • 架構：手機即算力（手機執行 AI/物理，眼鏡負責顯示 + 追蹤）
  • 誠實標籤：PWA 路徑已達參考就緒；未與 Meta 達成聯合行銷或預裝協議

第 2 階段 — 平台

Android XR + 原生 SDK · 2027 年

  • 入口：同一 PWA 路徑可推廣至 Android XR 的標準化 Chrome 表面
  • 原生 SDK：raku-runtime 以帶 OpenXR 相容工作階段層的原生 DLL 形式 — 接入 OEM 的合成器、手部追蹤管線或顯示驅動
  • SDK 文件目標：合作夥伴級文件 2026 年第四季；8–12 個開發週的整合路徑
  • 合作夥伴：Samsung Galaxy XR + Android XR 被授權方；Xreal、Rokid、Vision Pro 瀏覽器同享分階段方案
  • 誠實標籤：Android XR 發布窗口取決於平台日程；無已簽約 OEM

第 3 階段 — 擴散

Xreal / Rokid / Galaxy XR / Vision Pro · 2027 年+

  • 透過瀏覽器路徑支援長尾裝置
  • 面向低規格眼鏡的離線 / 間歇連結韌性
  • 可重複的自助資格 → PWA 試點 → SDK 管線
  • 為一家 Tier 3 策略合作夥伴打造聯合行銷發布應用

從眼鏡到 MCP，一圖說明。

眼鏡上的 LLM 透過 PWA 與 RakuAI 的 MCP 表面對話 — 經由到配套裝置的 stdio 橋，或經由 HTTPS 到託管 MCP。執行階段以空間原語作答：錨點、姿態、場景查詢、splat 控制代碼。

兩端都是真實且已上線的。生產 MCP 伺服器暴露 17 個原生工具（5 個唯讀 + 12 個變更）— 完整清單見 spatial-engine.html。其中與眼鏡相關的約 12 個劃分為：6 個核心擷取/場景工具（在 MCP 頁面有文件），以及 6 個 xr_ 轉接器工具（面向眼鏡驅動流程的工作階段、頭部姿態、輸入和錨點原語）。託管 MCP 執行在 East US 的 Azure Container Apps 上。
