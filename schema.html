.raku File Structure

A .raku file is a JSON document with 5 required sections.

1

raku_format

Schema version string. Currently "1.0.0".

2

metadata

Game info: name, description, genre (space_shooter, tower_defense, puzzle, platformer, arena_combat, etc.), creator, version, tags.

3

initialization

Engine setup calls. Must start with raku_init. Set up window, physics, audio, and AI subsystems.

4

calls

Gameplay setup: create scenes, load assets, bind input, configure AI. Must end with raku_scene_start_loop.

5

shutdown

Cleanup: destroy scenes and shut down engine. Must end with raku_shutdown.

// Paste this prompt into any AI assistant:

Generate a .raku game file for a space shooter game with these features:
- Adaptive difficulty that scales with player performance
- Keyboard controls (arrows + space to fire)
- Background music with spatial audio
- 1280x720 window

Use the Raku Runtime schema from https://rakuai.com/schema.json

All API functions follow the pattern: raku_<module>_<action>
