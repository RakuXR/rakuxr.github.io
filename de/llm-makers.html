Anmelden EN

Geben Sie Ihrer KI einen Körper in der realen Welt.

RakuAI ist die Runtime, in der Ihr Modell zu Hause ist, wenn es eine räumliche Welt bauen, simulieren oder mit ihr interagieren muss. Anbindung über MCP. Keine individuelle Integration. Von Grund auf herstellerübergreifend — Ihr Modell konkurriert über Qualität, nicht über die Integrationstiefe.

Über MCP anbinden → Die Architektur lesen →

Warum MCP-nativ für LLM-Hersteller wichtig ist

1. Ihr Modell funktioniert einfach.

Wenn Ihr Assistent MCP spricht, spricht er RakuAI. Sechs Tools. Derselbe Vertrag für jedes Modell. Keine individuelle Integration.

2. Von Grund auf herstellerübergreifend.

Claude, ChatGPT, Gemini, Copilot — sie alle steuern RakuAI auf dieselbe Weise. Ihr Modell konkurriert über die Qualität der Absicht.

3. Produktiv: Deny-by-Default.

Audit-protokolliert. Schema-validiert. Read-only-Tools laufen in der Produktion; Mutations-Tools in Sandbox/Dev.

Architektur

Was möglich ist

  • NPCs, die beobachten, bevor sie sprechen — Ihr Modell liest den Szenenzustand in jedem Frame
  • Erlebnisse, die sich um echte GPS-Koordinaten herum neu formen
  • World Models auf dem Simulationsschritt — Ihr Modell ist das Gehirn, kein Content-Lieferant
  • Plattformübergreifende Auslieferung — Smart Glasses, Headsets, Web, mobil

vs Unity

Unity hat gerade Native MCP veröffentlicht. Es eignet sich gut für die Szenenmanipulation zur Designzeit: Claude Desktop baut eine Szene, Unity rendert sie. Das MCP von RakuAI läuft zur Simulationszeit. Ihr Modell erzeugt keine Szenendatei — es steuert die Welt Frame für Frame. Andere Kategorie, beide gültig.

Adaptervertrag

Über die sechs Kern-Tools hinaus liefern Partner Adapter aus:

  • VeoEnvironmentAdapter — Googles Veo
  • RunwayEnvironmentAdapter — Runway
  • YourModelAdapter — Ihr Modell

Der Adaptervertrag ist öffentlich. Jedes Modell, das Frames + Szenen-Deltas erzeugt, kann zu einem Adapter werden.

Belege für herstellerübergreifendes Arbeiten

Unser Dev-Loop nutzt vier KI-Anbieter parallel:

  • Claude Code schreibt den Großteil unserer Runtime
  • Gemini prüft PRs
  • ChatGPT dient als Sparringspartner für die Architektur
  • GitHub Copilot fängt ab, was die anderen übersehen

Wir sind keinem einzelnen Modell treu. Unsere Runtime auch nicht.
