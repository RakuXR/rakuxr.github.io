Steuern Sie RakuAI aus Ihrem KI-Assistenten.

Aktualisiert (2026): Die MCP-Oberfläche umfasst inzwischen 17 native Tools (5 schreibgeschützt + 12 mutierend) plus universellen api_call/api_search-Passthrough.

Sechs MCP-Tools. Produktiv im Einsatz seit März 2026. Deny-by-Default-Berechtigungen. Vollständiges Audit-Log. Die Runtime, mit der Ihre KI spricht, wenn sie eine räumliche Welt bauen, abfragen oder simulieren muss.

Build-Log lesen → Auf GitHub ansehen →

So funktioniert es

Ihr KI-Assistent spricht MCP. RakuAI ist die Runtime am anderen Ende dieser Verbindung und steuert echte räumliche Ausgaben an Brillen, Headsets und den Browser.

Die sechs Tools

Die vollständige MCP-Schnittstelle. Schema-validiert, audit-protokolliert, berechtigungsgesteuert.

Tool Was es tut Berechtigung
load_world_model(adapter_name, config) Einen World-Model-Adapter registrieren Nur Sandbox/Dev
ingest_frame(adapter_name, frame_data, frame_index, timestamp) Einen Frame in den Szenengraph einspeisen Nur Sandbox/Dev

set_render_target(target_type, config) Ausgabe konfigurieren (WebGL/VR/nativ/offscreen) Nur Sandbox/Dev
start_simulation(tick_rate, max_duration, realtime) Simulationsschleife starten Nur Sandbox/Dev
get_metrics() Performance-Telemetrie Alle Modi

vs Unity MCP

Unity MCP (gerade veröffentlicht): Szenenmanipulation, Asset-Platzierung, Build-Automatisierung — Claude Desktop erzeugt statische Szenen zur Designzeit.

RakuAI MCP (produktiv seit März 2026): World-Model-Orchestrierung — jedes LLM streamt in jedem Frame Absichten in den Simulationsschritt.

Unity ist das Werkzeug, das Ihre KI in die Hand nimmt. RakuAI ist der Körper, in dem Ihre KI lebt.

Adapter

Der Vertrag sind die sechs Tools. Die Adapter sind die Art, wie Partner sich anbinden.

Eigener Adapter Ihr Modell

Der Adaptervertrag ist öffentlich. Bauen Sie einen und werden Sie Teil des Ökosystems.

Warum MCP-nativ wichtig ist

  1. Keine individuelle Integration — wenn Ihr Modell MCP spricht, spricht es RakuAI.
  2. Von Grund auf herstellerübergreifend — Claude, ChatGPT, Gemini, Copilot steuern alle dieselben sechs Tools.
  3. Produktionsreif — Deny-by-Default, audit-protokolliert, schema-validiert.
