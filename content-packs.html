Content Packs

Curated Assets for Every Template — two modes, one URI convention, zero guesswork.

Prototype vs Presentation

Every template ships with two content packs. Choose the one that fits your stage of development.

Prototype Mode

Lightweight placeholder assets designed for fast iteration. Simple shapes, solid colors, and basic sound effects so you can test gameplay logic without waiting for art. Ideal for game jams and early prototyping.

  • ✓ Minimal file sizes
  • ✓ Instant load times
  • ✓ Clear visual hierarchy
  • ✓ Consistent color coding

Presentation Mode

Polished, high-fidelity assets for demos, trailers, and showcases. Detailed textures, particle effects, adaptive music with stems, and professional UI elements. Ship-quality visuals out of the box.

  • ✓ HD textures & sprites
  • ✓ Multi-layer adaptive audio
  • ✓ Particle VFX library
  • ✓ Themed UI components

URI Convention

All content pack assets follow a single, predictable URI pattern. AI code generators and human developers use the same paths.

// URI structure
engine://packs/<template>/<mode>/<category>/<asset>

// Examples
engine://packs/space_shooter/prototype/textures/ship.png
engine://packs/space_shooter/presentation/vfx/explosion_01.vfx
engine://packs/platformer/presentation/audio/music/main_loop.ogg
engine://packs/tower_defense/prototype/sprites/tower_basic.png
engine://packs/racing/presentation/models/kart_01.glb

Categories: textures · sprites · materials · models · vfx · ui · audio/music · audio/stems · audio/sfx · audio/ui

Referencing Packs in .raku Files

Declare a content pack in metadata.content_pack, then use pack URIs wherever you reference assets.

1. Declare the content pack in metadata

// Inside your .raku file's metadata section
"metadata": {
  "name": "Galaxy Defender",
  "genre": "space_shooter",
  "content_pack": {
    "template": "space_shooter",
    "mode": "presentation",
    "pack_version": "0.1.0"
  }
}

2. Use pack URIs in asset references

// Load a texture from the content pack
{
  "function": "load_texture",
  "params": {
    "name": "ship_tex",
    "path": engine://packs/space_shooter/presentation/textures/ship.png
  }
}

// Load audio from the content pack
{
  "function": "load_audio",
  "params": {
    "name": "bgm_main",
    "path": engine://packs/space_shooter/presentation/audio/music/main_loop.ogg
  }
}

3. Declare an audio plan (optional)

// Map audio assets for adaptive music and SFX
"audio_plan": {
  "music": {
    "main_loop": engine://packs/space_shooter/presentation/audio/music/main_loop.ogg,
    "intensity_layers": [
      engine://packs/space_shooter/presentation/audio/stems/intensity_low.ogg,
      engine://packs/space_shooter/presentation/audio/stems/intensity_mid.ogg,
      engine://packs/space_shooter/presentation/audio/stems/intensity_high.ogg
    ]
  },
  "sfx": {
    "weapon_fire": engine://packs/space_shooter/presentation/audio/sfx/laser.ogg,
    "explosion": engine://packs/space_shooter/presentation/audio/sfx/explosion.ogg
  }
}

Audio Pack Pipeline

Content packs include a full audio pipeline with five asset categories that feed into the engine's adaptive audio system.

1

Music

Main loops and background tracks. Each pack includes at least one loopable music file tuned to the template's genre.

2

Stems

Intensity layers that the adaptive audio system mixes in real time based on gameplay tension (low, medium, high).

3

SFX

Gameplay sound effects: weapon fire, impacts, explosions, pickups, and warnings. Mapped via audio_plan.sfx.

4

UI Sounds

Interface audio: clicks, confirmations, and error tones. Referenced through audio_plan.ui.

5

Voice Hooks

Placeholder voice event markers for templates that support NPC dialogue or narrator systems.

Manifest Schema

Every content pack is described by a JSON manifest file that conforms to the Content Pack Manifest Schema. The manifest declares the pack's template, mode, version, asset lists, audio assets, and fallback URIs.

Required Fields

pack_id, template, mode, version, engine_version_min, mount_root, assets, audio, fallbacks

Asset Categories

Visual: textures, sprites, materials, models, vfx, ui. Audio: music, stems, sfx, ui, voice_hooks.

Fallbacks

Every manifest must declare missing_texture and missing_audio fallback URIs so the engine never hits a missing asset error.
